## 11.1 Nginx 이해

### Nginx vs Apache

| 특성 | Nginx | Apache |
|------|-------|--------|
| 아키텍처 | 이벤트 기반(비동기) | 프로세스/스레드 기반 |
| 동시 접속 | 매우 높음 | 상대적으로 낮음 |
| 메모리 | 낮은 메모리 사용 | 높은 메모리 사용 |
| 정적 파일 | 매우 빠름 | 빠름 |
| 설정 | 중앙 집중식 | 분산(.htaccess) |
| 모듈 | 컴파일 타임 | 런타임 로드 가능 |

### Nginx 설치

```bash
# 공식 저장소에서 설치
sudo apt update
sudo apt install nginx

# 최신 버전 (Nginx 공식 저장소)
curl -fsSL https://nginx.org/keys/nginx_signing.key | \
  sudo gpg --dearmor -o /usr/share/keyrings/nginx-keyring.gpg

echo "deb [signed-by=/usr/share/keyrings/nginx-keyring.gpg] \
  http://nginx.org/packages/ubuntu $(lsb_release -cs) nginx" | \
  sudo tee /etc/apt/sources.list.d/nginx.list

sudo apt update && sudo apt install nginx

# 서비스 시작 및 활성화
sudo systemctl enable --now nginx

# 버전 확인
nginx -v
```

## 11.2 Nginx 설정 파일 구조

```
/etc/nginx/
├── nginx.conf              ← 메인 설정 파일
├── conf.d/                 ← 추가 설정 (자동 로드)
│   └── default.conf
├── sites-available/        ← 가상 호스트 설정 (Ubuntu 방식)
│   └── default
├── sites-enabled/          ← 활성화된 가상 호스트 (심볼릭 링크)
│   └── default -> ../sites-available/default
├── snippets/               ← 재사용 설정 조각
│   ├── fastcgi-php.conf
│   └── snakeoil.conf
└── modules-enabled/        ← 활성화된 모듈
```

### nginx.conf 구조

```nginx
# /etc/nginx/nginx.conf

user www-data;
worker_processes auto;          # CPU 코어 수
pid /run/nginx.pid;

events {
    worker_connections 1024;    # 워커당 최대 연결 수
    use epoll;                  # Linux 이벤트 모델
    multi_accept on;
}

http {
    # MIME 유형
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    # 로그 형식
    log_format main '$remote_addr - $remote_user [$time_local] '
                    '"$request" $status $body_bytes_sent '
                    '"$http_referer" "$http_user_agent"';

    access_log /var/log/nginx/access.log main;
    error_log /var/log/nginx/error.log warn;

    # 성능 설정
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    gzip on;

    # 가상 호스트 설정 포함
    include /etc/nginx/conf.d/*.conf;
    include /etc/nginx/sites-enabled/*;
}
```

## 11.3 가상 호스트 설정

### 기본 정적 사이트

```bash
# 사이트 디렉토리 생성
sudo mkdir -p /var/www/example.com/html
sudo chown -R $USER:$USER /var/www/example.com/html

# 테스트 페이지 생성
cat > /var/www/example.com/html/index.html << 'EOF'
<!DOCTYPE html>
<html>
<body><h1>example.com 동작 중!</h1></body>
</html>
EOF
```

```bash
# 가상 호스트 설정 파일 생성
sudo nano /etc/nginx/sites-available/example.com
```

```nginx
server {
    listen 80;
    listen [::]:80;

    server_name example.com www.example.com;
    root /var/www/example.com/html;
    index index.html index.htm;

    # 액세스/에러 로그
    access_log /var/log/nginx/example.com.access.log;
    error_log  /var/log/nginx/example.com.error.log;

    location / {
        try_files $uri $uri/ =404;
    }

    # 정적 파일 캐싱
    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 1y;
        add_header Cache-Control "public, no-transform";
    }

    # 숨김 파일 차단
    location ~ /\. {
        deny all;
    }
}
```

```bash
# 가상 호스트 활성화
sudo ln -s /etc/nginx/sites-available/example.com /etc/nginx/sites-enabled/

# 설정 문법 검사
sudo nginx -t

# Nginx 재로드
sudo systemctl reload nginx
```

## 11.4 HTTPS 설정 (Let's Encrypt)

### Certbot 설치

```bash
# Certbot 설치
sudo apt install certbot python3-certbot-nginx

# 인증서 발급 (도메인이 실제로 가리키고 있어야 함)
sudo certbot --nginx -d example.com -d www.example.com

# 와일드카드 인증서
sudo certbot certonly --manual -d "*.example.com" \
  --preferred-challenges dns
```

### 수동 SSL 설정

```nginx
server {
    listen 80;
    server_name example.com www.example.com;
    # HTTP → HTTPS 리다이렉트
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name example.com www.example.com;

    root /var/www/example.com/html;
    index index.html;

    # SSL 인증서
    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    # SSL 보안 설정
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers off;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # HSTS (HTTPS 강제)
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # 보안 헤더
    add_header X-Frame-Options DENY;
    add_header X-Content-Type-Options nosniff;
    add_header X-XSS-Protection "1; mode=block";

    location / {
        try_files $uri $uri/ =404;
    }
}
```

```bash
# 자동 갱신 테스트
sudo certbot renew --dry-run

# cron으로 자동 갱신 (certbot이 자동 설정)
sudo systemctl status certbot.timer
```

## 11.5 리버스 프록시

### Node.js/Python 앱 연결

```nginx
# /etc/nginx/sites-available/myapp
server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 60s;
    }
}
```

### 업스트림 로드밸런싱

```nginx
upstream myapp {
    least_conn;                          # 최소 연결 방식
    server 127.0.0.1:3001 weight=3;     # 가중치
    server 127.0.0.1:3002 weight=1;
    server 127.0.0.1:3003 backup;       # 백업 서버
    keepalive 32;
}

server {
    listen 80;
    location / {
        proxy_pass http://myapp;
    }
}
```

## 11.6 성능 최적화

```nginx
# /etc/nginx/conf.d/performance.conf

# Gzip 압축
gzip on;
gzip_vary on;
gzip_proxied any;
gzip_comp_level 6;
gzip_types text/plain text/css text/xml application/json
           application/javascript application/xml+rss
           application/atom+xml image/svg+xml;

# 파일 캐시
open_file_cache max=1000 inactive=20s;
open_file_cache_valid 30s;
open_file_cache_min_uses 2;

# 버퍼 설정
client_body_buffer_size 128k;
client_max_body_size 10m;
client_header_buffer_size 1k;
```

```bash
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -10
```

## 핵심 요약

- **설치 후 필수**: `systemctl enable nginx`, `ufw allow 'Nginx Full'`
- **설정 구조**: sites-available에 작성 → sites-enabled에 심볼릭 링크
- **`nginx -t`**: 설정 변경 후 항상 문법 검사
- **Let's Encrypt**: certbot으로 무료 HTTPS 인증서 발급
- **리버스 프록시**: Node.js/Python 앱 앞에 Nginx 배치
- **보안 헤더**: HSTS, X-Frame-Options 등 필수 설정
