# 챕터 4: 파일시스템 구조와 관리

> 리눅스 파일시스템의 구조를 이해하고 디스크를 관리한다

## 학습 목표
- 리눅스 FHS(Filesystem Hierarchy Standard)를 이해한다
- 디스크 파티션과 마운트 개념을 익힌다
- 디스크 사용량을 확인하고 관리할 수 있다
- 심볼릭 링크와 하드 링크를 이해한다

## 4.1 리눅스 디렉토리 구조 (FHS)

### 최상위 디렉토리 구조

```
/ (루트)
├── bin/      → 기본 명령어 바이너리 (ls, cp, mv 등)
├── sbin/     → 시스템 관리 명령어 (fdisk, mkfs 등)
├── boot/     → 부트로더, 커널 이미지
├── dev/      → 장치 파일 (디스크, 터미널 등)
├── etc/      → 시스템 설정 파일
├── home/     → 사용자 홈 디렉토리
├── lib/      → 공유 라이브러리
├── media/    → 이동식 미디어 마운트 포인트
├── mnt/      → 임시 마운트 포인트
├── opt/      → 선택적 소프트웨어 패키지
├── proc/     → 프로세스/커널 정보 (가상 파일시스템)
├── root/     → root 사용자 홈 디렉토리
├── run/      → 런타임 데이터
├── srv/      → 서비스 데이터
├── sys/      → 시스템/커널 정보 (가상 파일시스템)
├── tmp/      → 임시 파일 (재부팅 시 삭제)
├── usr/      → 사용자 프로그램과 데이터
│   ├── bin/      → 사용자 명령어
│   ├── lib/      → 라이브러리
│   ├── local/    → 로컬 설치 소프트웨어
│   └── share/    → 공유 데이터
└── var/      → 가변 데이터 (로그, 캐시, 스풀)
    ├── log/      → 시스템 로그
    ├── cache/    → 캐시 데이터
    └── www/      → 웹 서버 파일
```

### 주요 디렉토리 상세 설명

```bash
# /etc - 시스템 설정 파일
/etc/passwd         # 사용자 계정 정보
/etc/shadow         # 비밀번호 (암호화)
/etc/group          # 그룹 정보
/etc/hosts          # 호스트명-IP 매핑
/etc/fstab          # 파일시스템 마운트 설정
/etc/crontab        # 예약 작업
/etc/ssh/           # SSH 설정
/etc/apt/           # APT 패키지 관리자 설정

# /proc - 프로세스 정보 (메모리에 존재)
cat /proc/cpuinfo   # CPU 정보
cat /proc/meminfo   # 메모리 정보
cat /proc/version   # 커널 버전
cat /proc/1/status  # PID 1번 프로세스 상태

# /dev - 장치 파일
/dev/sda            # 첫 번째 SATA 디스크
/dev/sda1           # 첫 번째 파티션
/dev/nvme0n1        # NVMe SSD
/dev/tty            # 터미널
/dev/null           # 블랙홀 (버리기)
/dev/zero           # 0 무한 제공
/dev/random         # 난수 생성
```

## 4.2 리눅스 파일시스템 유형

### 주요 파일시스템

| 파일시스템 | 특징 | 용도 |
|-----------|------|------|
| **ext4** | 리눅스 표준, 안정적 | 일반 용도 기본 |
| **xfs** | 대용량, 고성능 | 대규모 파일 서버 |
| **btrfs** | 스냅샷, 압축 지원 | 현대적 기능 필요 시 |
| **tmpfs** | 메모리 기반 | /tmp, /run |
| **vfat** | FAT32 호환 | USB, 부팅 파티션 |
| **ntfs** | Windows 호환 | Windows 파티션 읽기 |

### 파일시스템 확인

```bash
# 파티션과 파일시스템 확인
lsblk
lsblk -f

# 마운트된 파일시스템 목록
df -h
findmnt

# 파일시스템 유형 확인
df -T
blkid /dev/sda1
```

## 4.3 디스크 파티션 관리

### 파티션 구조

```
물리 디스크 (/dev/sda)
┌────────────────────────────────────────┐
│ MBR/GPT │ 파티션1  │ 파티션2  │ 파티션3 │
│         │ /dev/sda1│ /dev/sda2│/dev/sda3│
│         │ /boot    │ swap     │ /       │
└────────────────────────────────────────┘
```

### fdisk로 파티션 관리

```bash
# 디스크 목록 확인
sudo fdisk -l

# 특정 디스크 파티션 확인
sudo fdisk -l /dev/sdb

# 파티션 작업 (대화형)
sudo fdisk /dev/sdb
# p: 파티션 목록
# n: 새 파티션 생성
# d: 파티션 삭제
# t: 파티션 유형 변경
# w: 저장 후 종료
# q: 저장 없이 종료
```

### parted로 파티션 관리 (GPT 권장)

```bash
# GPT 파티션 테이블 생성
sudo parted /dev/sdb mklabel gpt

# 파티션 생성
sudo parted /dev/sdb mkpart primary ext4 0% 100%

# 파티션 목록 확인
sudo parted /dev/sdb print
```

### 파일시스템 생성 (포맷)

```bash
# ext4 파일시스템 생성
sudo mkfs.ext4 /dev/sdb1

# xfs 파일시스템 생성
sudo mkfs.xfs /dev/sdb1

# 레이블 지정
sudo mkfs.ext4 -L "mydata" /dev/sdb1
```

## 4.4 마운트와 언마운트

### 마운트 개념

```
장치 파일         마운트 포인트      접근 경로
/dev/sdb1  →  /mnt/data 로 마운트 →  /mnt/data/files
```

### 임시 마운트

```bash
# 마운트 포인트 생성
sudo mkdir -p /mnt/data

# 마운트
sudo mount /dev/sdb1 /mnt/data

# 마운트 확인
mount | grep sdb1
df -h /mnt/data

# 언마운트
sudo umount /mnt/data
# 또는
sudo umount /dev/sdb1

# 강제 언마운트 (사용 중일 때)
sudo umount -f /mnt/data
sudo umount -l /mnt/data  # lazy unmount
```

### 영구 마운트 (/etc/fstab)

```bash
# UUID 확인 (UUID 사용 권장, 디바이스명 변경 가능)
sudo blkid /dev/sdb1

# /etc/fstab 편집
sudo nano /etc/fstab
```

```
# /etc/fstab 형식:
# <장치>                <마운트포인트>  <유형>  <옵션>         <dump> <pass>
UUID=xxxxxxxx-xxxx...  /mnt/data       ext4    defaults        0      2
/dev/sdb1              /mnt/backup     xfs     defaults,noatime 0     2
tmpfs                  /tmp            tmpfs   defaults,size=1G 0     0
```

```bash
# fstab 설정 테스트
sudo mount -a

# 특정 항목만 마운트
sudo mount /mnt/data
```

## 4.5 디스크 사용량 확인

### df - 파일시스템 사용량

```bash
# 기본 출력
df -h

# 특정 디렉토리
df -h /home

# 파일시스템 유형 포함
df -hT

# inode 사용량
df -i
```

```
출력 예:
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        20G  8.5G  10G  46%  /
tmpfs           2.0G  1.2M  2.0G   1% /tmp
/dev/sdb1        50G   12G  38G  24%  /mnt/data
```

### du - 디렉토리 사용량

```bash
# 현재 디렉토리 전체 크기
du -sh .

# 하위 디렉토리별 크기
du -h --max-depth=1 /var

# 상위 10개 큰 디렉토리
du -h /var | sort -rh | head -10

# 특정 패턴 제외
du -sh --exclude=*.log /var/log

# 파일별 크기
du -ah /etc | sort -rh | head -20
```

### ncdu - 대화형 디스크 사용량 분석

```bash
# 설치
sudo apt install ncdu

# 실행
ncdu /
ncdu /home

# 조작: j/k 또는 화살표로 이동, d로 삭제, q로 종료
```

## 4.6 스왑(Swap) 관리

### 스왑 파일 생성

```bash
# 2GB 스왑 파일 생성
sudo fallocate -l 2G /swapfile

# 권한 설정
sudo chmod 600 /swapfile

# 스왑 파일시스템 설정
sudo mkswap /swapfile

# 스왑 활성화
sudo swapon /swapfile

# 스왑 확인
swapon --show
free -h
```

```bash
# 영구 설정 (/etc/fstab)
/swapfile  none  swap  sw  0  0
```

## 4.7 링크 (Link)

### 하드 링크 vs 심볼릭 링크

```
하드 링크:
원본파일 ──┐
           ├── inode (실제 데이터)
하드링크 ──┘

심볼릭 링크:
심볼릭링크 → 원본파일경로 → inode (실제 데이터)
```

### 하드 링크

```bash
# 하드 링크 생성
ln original.txt hardlink.txt

# 확인 (inode 번호 동일)
ls -li original.txt hardlink.txt

# 특징
# - 같은 파일시스템만 가능
# - 디렉토리 불가
# - 원본 삭제해도 데이터 유지
```

### 심볼릭 링크

```bash
# 심볼릭 링크 생성
ln -s /path/to/original symlink

# 현재 디렉토리에 생성
ln -s /usr/local/bin/python3.11 python3

# 확인
ls -la symlink
# symlink -> /path/to/original

# 링크 삭제 (원본 유지)
rm symlink
unlink symlink

# 끊어진 링크 찾기
find . -type l ! -e
```

### 실용적인 심볼릭 링크 활용

```bash
# 버전 관리
ln -s /opt/python3.11 /usr/local/bin/python3

# 설정 파일 관리
ln -s ~/dotfiles/.bashrc ~/.bashrc

# 웹 서버 공개 디렉토리
sudo ln -s /var/www/myapp/public /var/www/html/myapp
```

## 4.8 파일시스템 점검과 복구

```bash
# 파일시스템 점검 (언마운트 상태에서)
sudo fsck /dev/sdb1
sudo fsck.ext4 -f /dev/sdb1

# ext4 파일시스템 정보 확인
sudo tune2fs -l /dev/sda1

# 저널 로그 확인
sudo dumpe2fs /dev/sda1 | grep -i journal
```

## 핵심 요약

- **FHS**: / 하위의 bin, etc, home, var 등 표준 디렉토리 구조
- **파티션**: fdisk(MBR), parted(GPT)로 관리
- **마운트**: mount 명령어, /etc/fstab으로 영구 설정
- **df**: 파일시스템 전체 사용량, **du**: 디렉토리별 사용량
- **심볼릭 링크**: 경로의 별칭, 버전 관리에 유용
- **하드 링크**: 동일 inode를 가리키는 또 다른 이름
