# 챕터 9: 방화벽과 보안

> UFW와 iptables로 서버를 안전하게 보호한다

## 학습 목표
- UFW로 방화벽을 설정하고 관리할 수 있다
- iptables 기본 개념을 이해한다
- fail2ban으로 브루트포스 공격을 방어할 수 있다
- 리눅스 보안 모범 사례를 적용할 수 있다

## 9.1 UFW (Uncomplicated Firewall)

### UFW 기본 설정

```bash
# UFW 상태 확인
sudo ufw status
sudo ufw status verbose
sudo ufw status numbered

# UFW 활성화/비활성화
sudo ufw enable
sudo ufw disable

# 기본 정책 설정 (활성화 전에 먼저 설정)
sudo ufw default deny incoming    # 들어오는 모든 연결 거부
sudo ufw default allow outgoing   # 나가는 모든 연결 허용
```

### 규칙 추가

```bash
# 서비스 이름으로 허용
sudo ufw allow ssh
sudo ufw allow http
sudo ufw allow https

# 포트 번호로 허용
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 8080

# UDP 허용
sudo ufw allow 53/udp      # DNS
sudo ufw allow 123/udp     # NTP

# 포트 범위
sudo ufw allow 6000:6007/tcp

# 특정 IP 허용
sudo ufw allow from 192.168.1.100
sudo ufw allow from 192.168.1.0/24

# 특정 IP + 포트
sudo ufw allow from 192.168.1.100 to any port 3306

# 특정 인터페이스
sudo ufw allow in on ens33 to any port 80
```

### 규칙 삭제

```bash
# 번호로 삭제
sudo ufw status numbered
sudo ufw delete 3

# 규칙 내용으로 삭제
sudo ufw delete allow 8080
sudo ufw delete allow from 192.168.1.100

# 전체 규칙 초기화
sudo ufw reset
```

### UFW 로그

```bash
# 로그 활성화
sudo ufw logging on
sudo ufw logging medium   # low/medium/high/full

# 로그 확인
sudo tail -f /var/log/ufw.log
sudo grep "UFW BLOCK" /var/log/ufw.log | head -20
```

## 9.2 iptables 기초

### iptables 구조

```
패킷 흐름:
                    INPUT
                      ↑
외부 → PREROUTING → 라우팅 → FORWARD → 라우팅 → POSTROUTING → 외부
                      ↓
                   OUTPUT
                      ↑
                  로컬 프로세스

테이블:
- filter: 패킷 허용/차단 (기본)
- nat: 주소 변환 (NAT)
- mangle: 패킷 수정
- raw: 추적 예외

체인:
- INPUT: 서버로 들어오는 패킷
- OUTPUT: 서버에서 나가는 패킷
- FORWARD: 서버를 통과하는 패킷
- PREROUTING: 라우팅 전 처리
- POSTROUTING: 라우팅 후 처리
```

### iptables 기본 명령어

```bash
# 현재 규칙 확인
sudo iptables -L
sudo iptables -L -v -n        # 자세히, 숫자 표시
sudo iptables -L INPUT -v -n

# 기본 정책 설정
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# 규칙 추가 (-A: 뒤에 추가, -I: 앞에 삽입)
# 루프백 허용
sudo iptables -A INPUT -i lo -j ACCEPT

# 기존 연결 유지
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# SSH 허용
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# HTTP/HTTPS 허용
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# ICMP (ping) 허용
sudo iptables -A INPUT -p icmp -j ACCEPT

# 규칙 삭제
sudo iptables -D INPUT -p tcp --dport 8080 -j ACCEPT

# 규칙 저장 (재부팅 후에도 유지)
sudo apt install iptables-persistent
sudo netfilter-persistent save
```

### iptables 실전 예제

```bash
# SSH 브루트포스 방어 (분당 3회 제한)
sudo iptables -A INPUT -p tcp --dport 22 -m state --state NEW \
  -m recent --set --name SSH
sudo iptables -A INPUT -p tcp --dport 22 -m state --state NEW \
  -m recent --update --seconds 60 --hitcount 4 --name SSH -j DROP

# 특정 IP 차단
sudo iptables -A INPUT -s 1.2.3.4 -j DROP

# 포트 스캔 차단
sudo iptables -A INPUT -p tcp --tcp-flags ALL NONE -j DROP
sudo iptables -A INPUT -p tcp --tcp-flags ALL ALL -j DROP

# DDoS 기본 방어
sudo iptables -A INPUT -p tcp --dport 80 -m limit \
  --limit 25/minute --limit-burst 100 -j ACCEPT
```

## 9.3 fail2ban

### 설치와 기본 설정

```bash
# 설치
sudo apt install fail2ban

# 서비스 시작
sudo systemctl enable --now fail2ban

# 상태 확인
sudo fail2ban-client status
sudo fail2ban-client status sshd
```

### fail2ban 설정

```bash
# 설정 파일 구조
/etc/fail2ban/jail.conf       # 기본 설정 (수정 금지)
/etc/fail2ban/jail.local      # 커스텀 설정 (여기서 수정)
/etc/fail2ban/filter.d/       # 필터 정의 파일
/etc/fail2ban/action.d/       # 액션 정의 파일

# jail.local 생성
sudo nano /etc/fail2ban/jail.local
```

```ini
[DEFAULT]
# 기본 설정
bantime  = 3600          # 차단 시간 (초), -1은 영구
findtime = 600           # 이 시간 내에
maxretry = 5             # 이 횟수 실패 시 차단
ignoreip = 127.0.0.1/8 192.168.1.0/24  # 화이트리스트

# 이메일 알림 (선택)
destemail = admin@example.com
sendername = Fail2Ban
action = %(action_mwl)s

[sshd]
enabled  = true
port     = ssh
filter   = sshd
logpath  = /var/log/auth.log
maxretry = 3
bantime  = 86400         # 24시간

[nginx-http-auth]
enabled  = true
filter   = nginx-http-auth
port     = http,https
logpath  = /var/log/nginx/error.log
maxretry = 5

[nginx-limit-req]
enabled = true
filter  = nginx-limit-req
port    = http,https
logpath = /var/log/nginx/error.log
maxretry = 10
```

```bash
# 설정 적용
sudo systemctl restart fail2ban

# 차단된 IP 확인
sudo fail2ban-client status sshd

# IP 차단 해제
sudo fail2ban-client set sshd unbanip 1.2.3.4

# 수동 IP 차단
sudo fail2ban-client set sshd banip 1.2.3.4
```

## 9.4 보안 업데이트 자동화

### unattended-upgrades

```bash
# 설치
sudo apt install unattended-upgrades

# 설정
sudo nano /etc/apt/apt.conf.d/50unattended-upgrades
```

```
Unattended-Upgrade::Allowed-Origins {
    "${distro_id}:${distro_codename}";
    "${distro_id}:${distro_codename}-security";
    "${distro_id}ESMApps:${distro_codename}-apps-security";
};

Unattended-Upgrade::Mail "admin@example.com";
Unattended-Upgrade::Remove-Unused-Dependencies "true";
Unattended-Upgrade::Automatic-Reboot "false";
```

```bash
# 자동 업데이트 주기 설정
sudo nano /etc/apt/apt.conf.d/20auto-upgrades
```

```
APT::Periodic::Update-Package-Lists "1";      # 매일 목록 업데이트
APT::Periodic::Download-Upgradeable-Packages "1";
APT::Periodic::AutocleanInterval "7";          # 7일마다 정리
APT::Periodic::Unattended-Upgrade "1";         # 자동 설치
```

## 9.5 시스템 보안 점검

### 보안 감사 도구

```bash
# lynis - 시스템 보안 감사
sudo apt install lynis
sudo lynis audit system

# rkhunter - 루트킷 검사
sudo apt install rkhunter
sudo rkhunter --update
sudo rkhunter --check

# chkrootkit
sudo apt install chkrootkit
sudo chkrootkit

# clamav - 바이러스 검사
sudo apt install clamav
sudo freshclam            # 데이터베이스 업데이트
sudo clamscan -r /home
```

### 열린 포트와 서비스 점검

```bash
# 불필요한 서비스 확인
sudo systemctl list-units --type=service --state=running

# 리스닝 포트 확인
sudo ss -tulnp

# setuid 파일 검사
sudo find / -perm /4000 -type f 2>/dev/null

# 최근 로그인 이력
last
lastb                     # 실패한 로그인
sudo grep "Failed" /var/log/auth.log | tail -20
sudo grep "Accepted" /var/log/auth.log | tail -20
```

## 9.6 AppArmor

```bash
# AppArmor 상태 확인
sudo apparmor_status
sudo aa-status

# 프로필 목록
ls /etc/apparmor.d/

# 특정 프로그램 프로필 확인

sudo dmesg | grep apparmor
```

## 핵심 요약

```
기본 보안 체크리스트:
□ sudo ufw enable && ufw default deny incoming
□ SSH: PermitRootLogin no, PasswordAuthentication no
□ SSH 포트 변경 (22 → 다른 포트)
□ fail2ban 설치 및 설정
□ 불필요한 서비스 비활성화
□ unattended-upgrades 보안 업데이트 자동화
□ 정기적인 lynis 보안 감사
□ 로그인 이력 주기적 확인
```

- **UFW**: iptables의 쉬운 인터페이스, `default deny incoming`이 기본
- **iptables**: 세밀한 패킷 필터링, 규칙 순서가 중요
- **fail2ban**: 로그 분석으로 브루트포스 자동 차단
- **보안 = 계층 방어**: 방화벽 + SSH 보안 + fail2ban + 업데이트
