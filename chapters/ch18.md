## 학습 목표

- 디스크와 파일시스템 문제를 해결할 수 있다

## 18.1 부팅 문제 해결

### 부팅 과정 이해

```
전원 켜짐
  ↓
BIOS/UEFI (하드웨어 초기화)
  ↓
GRUB (부트로더)
  ↓
Linux 커널 로드
  ↓
initramfs (초기 RAM 파일시스템)
  ↓
systemd (PID 1)
  ↓
타깃 (multi-user.target 등)
  ↓
로그인 프롬프트
```

### GRUB 복구

```bash
# GRUB 메뉴 진입 (부팅 시 ESC 또는 Shift)
# 복구 모드 선택: "Advanced options" → "recovery mode"

# 복구 모드 옵션:
# - clean: 디스크 공간 확보
# - dpkg: 패키지 데이터베이스 수정
# - fsck: 파일시스템 검사
# - root: root 쉘 열기

# GRUB 재설치 (Live USB에서)
sudo mount /dev/sda1 /mnt
sudo mount --bind /dev /mnt/dev
sudo mount --bind /proc /mnt/proc
sudo mount --bind /sys /mnt/sys
sudo chroot /mnt
grub-install /dev/sda
update-grub
exit
```

### systemd 부팅 문제 분석

```bash
# 부팅 시간 분석
systemd-analyze
systemd-analyze blame | head -20
systemd-analyze critical-chain

# 실패한 유닛 확인
systemctl --failed

# 특정 서비스 로그 확인
journalctl -u nginx --boot
journalctl -b -1 -p err      # 이전 부팅의 오류

# 비상 모드 (emergency target)
# GRUB에서 커널 파라미터에 추가:
# systemd.unit=emergency.target

# 단일 사용자 모드
# 커널 파라미터: single
```

## 18.2 디스크 문제 해결

### 디스크 꽉 참 (Disk Full)

```bash
# 원인 찾기
df -h                              # 어느 파티션이 꽉 찼는지
du -sh /* 2>/dev/null | sort -rh | head -10  # 큰 디렉토리

# 큰 파일 찾기
find / -size +100M -type f 2>/dev/null
find /var/log -name "*.log" -size +50M

# 로그 파일 정리
sudo journalctl --vacuum-size=200M
sudo apt clean
sudo apt autoremove

# 삭제됐지만 열려있는 파일 (디스크 공간 미반환)
lsof +L1 | awk 'NR>1 {print $7, $9}' | sort -rn | head -10
# 해당 프로세스 재시작 필요
```

### 파일시스템 오류 수정

```bash
# 파일시스템 마운트 해제 후 검사
sudo umount /dev/sdb1

# ext4 검사 및 수정
sudo fsck -f /dev/sdb1      # -f: 강제 검사
sudo fsck -y /dev/sdb1      # -y: 모든 오류 자동 수정
sudo e2fsck -f /dev/sda1

# XFS 수정
sudo xfs_repair /dev/sdb1

# 루트 파티션 검사 (재부팅 시)
sudo touch /forcefsck       # 다음 부팅 시 자동 검사
```

### inode 소진

```bash
# inode 사용량 확인
df -i

# 많은 파일이 있는 디렉토리 찾기
find / -xdev -printf '%h\n' 2>/dev/null | \
  sort | uniq -c | sort -rn | head -10

# 임시 파일 정리
find /tmp -type f -mtime +7 -delete
find /var/tmp -type f -mtime +30 -delete
```

## 18.3 네트워크 문제 진단

### 체계적 진단 순서

```
1. 물리 연결 확인
   ↓
2. IP 주소 확인
   ↓
3. 게이트웨이 ping
   ↓
4. DNS 확인
   ↓
5. 외부 연결 테스트
   ↓
6. 방화벽 규칙 확인
   ↓
7. 서비스 포트 확인
```

```bash
# 1. 인터페이스 확인
ip link show                    # UP 상태인지 확인
ip addr show

# 2. IP 주소 확인
ip addr | grep inet

# 3. 게이트웨이 ping
ip route | grep default         # 게이트웨이 IP 확인
ping -c 3 <gateway_ip>

# 4. DNS 확인
cat /etc/resolv.conf
dig google.com @8.8.8.8         # 직접 DNS 서버 지정
nslookup google.com

# 5. 외부 연결
ping -c 3 8.8.8.8              # IP 직접 (DNS 없이)
curl -v https://google.com

# 6. 방화벽 확인
sudo ufw status
sudo iptables -L -n

# 7. 포트 확인
ss -tlnp | grep :80
telnet localhost 80
nc -zv localhost 80
```

### 자주 발생하는 네트워크 문제

```bash
# DNS 캐시 초기화
sudo systemd-resolve --flush-caches
sudo resolvectl flush-caches

# 네트워크 재시작
sudo systemctl restart NetworkManager    # 데스크톱
sudo netplan apply                       # 서버

# 인터페이스 재시작
sudo ip link set ens33 down
sudo ip link set ens33 up
sudo dhclient ens33                      # DHCP 재할당

# MTU 문제 (패킷 분할 이슈)
ping -M do -s 1472 192.168.1.1           # MTU 테스트
sudo ip link set ens33 mtu 1400          # MTU 조정
```

## 18.4 서비스 문제 해결

### 서비스가 시작되지 않을 때

```bash
# 상태 확인 (가장 먼저)
sudo systemctl status nginx
# → Active: failed 또는 오류 메시지 확인

# 상세 로그 확인
sudo journalctl -u nginx -n 50 --no-pager
sudo journalctl -xe

# 설정 파일 문법 검사
sudo nginx -t
sudo apache2ctl configtest
sudo sshd -t

# 포트 충돌 확인
sudo ss -tlnp | grep :80
sudo fuser 80/tcp

# 프로세스 강제 종료 후 재시작
sudo fuser -k 80/tcp
sudo systemctl start nginx
```

### 프로세스 충돌 디버깅

```bash
# 코어 덤프 활성화
ulimit -c unlimited
echo "/tmp/core.%e.%p" | sudo tee /proc/sys/kernel/core_pattern

# gdb로 코어 분석
sudo apt install gdb
gdb /usr/bin/nginx /tmp/core.nginx.1234
(gdb) bt                    # 백트레이스

# strace로 시스템 콜 추적
sudo strace -p <PID> -e trace=network
sudo strace nginx 2>&1 | head -50
```

## 18.5 성능 튜닝

### 커널 파라미터 튜닝

```bash
# 현재 값 확인
sysctl -a | grep net.ipv4.tcp

# 임시 변경
sudo sysctl -w net.ipv4.tcp_fin_timeout=30

# 영구 변경
sudo nano /etc/sysctl.d/99-tuning.conf
```

```ini
# /etc/sysctl.d/99-tuning.conf

# 네트워크 성능
net.ipv4.tcp_fin_timeout = 30
net.ipv4.tcp_keepalive_time = 1200
net.core.somaxconn = 65535
net.ipv4.ip_local_port_range = 10000 65000
net.ipv4.tcp_max_syn_backlog = 8096

# 파일 디스크립터 제한
fs.file-max = 100000
fs.nr_open = 100000

# 가상 메모리
vm.swappiness = 10          # 스왑 사용 최소화
vm.dirty_ratio = 15
```

```bash
# 적용
sudo sysctl -p /etc/sysctl.d/99-tuning.conf
```

### 프로세스 파일 제한

```bash
# 현재 제한 확인
ulimit -n                       # 열 수 있는 파일 수
cat /proc/sys/fs/file-max

# 특정 서비스 제한 변경
sudo nano /etc/systemd/system/nginx.service.d/override.conf
```

```ini
[Service]
LimitNOFILE=65535
LimitNPROC=65535
```

## 18.6 실전 시나리오

### 시나리오 1: 웹서버 응답 없음

```bash
# 1. 서비스 상태 확인
sudo systemctl status nginx

# 2. 포트 리스닝 확인
ss -tlnp | grep :80

# 3. 방화벽 확인
sudo ufw status

# 4. Nginx 설정 확인
sudo nginx -t

# 5. 오류 로그 확인
sudo tail -f /var/log/nginx/error.log

# 6. 리소스 확인 (CPU/메모리)
top

# 7. 연결 수 확인
ss -tn | grep :80 | wc -l
```

### 시나리오 2: 서버 속도 저하

```bash
# 1. 부하 확인
uptime                          # load average 확인
# 1분 5분 15분 평균 (코어 수 초과면 과부하)

# 2. CPU/메모리 확인
top -b -n 1 | head -20

# 3. I/O 대기 확인
iostat -x 1 3
iotop -o -n 3

# 4. 메모리 부족 확인
free -h
vmstat 1 5

# 5. 네트워크 포화 확인
sar -n DEV 1 5

# 6. 원인 프로세스 확인
ps aux --sort=-%cpu | head -10
ps aux --sort=-%mem | head -10
```

## 핵심 요약

```
트러블슈팅 5단계:
1. 증상 파악      → 무엇이 안 되는가?
2. 정보 수집      → 로그, 상태 확인
3. 가설 수립      → 원인 추정
4. 검증           → 하나씩 테스트
5. 해결 및 기록   → 재발 방지
```

```
자주 쓰는 트러블슈팅 명령어:
systemctl status [서비스]       ← 서비스 상태
journalctl -u [서비스] -n 50    ← 서비스 로그
ss -tlnp                        ← 포트 확인
df -h && free -h                ← 디스크/메모리
top                             ← CPU/프로세스
sudo ufw status                 ← 방화벽
ip addr && ip route             ← 네트워크
```
