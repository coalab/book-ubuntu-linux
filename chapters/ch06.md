## 6.1 프로세스 이해

### 프로세스란?

```
프로그램 (디스크의 파일)
        ↓  실행 (exec)
프로세스 (메모리에 로드된 실행 중인 프로그램)

프로세스 구성 요소:
┌────────────────────────┐
│ PID (프로세스 ID)       │
│ PPID (부모 프로세스 ID) │
│ UID/GID (소유자/그룹)   │
│ 상태 (Running/Sleep..)  │
│ 메모리 공간             │
│ 열린 파일 디스크립터    │
│ 환경 변수              │
└────────────────────────┘
```

### 프로세스 상태

```
프로세스 상태 전이:
                  fork()
새 프로세스 ─────────────→ READY (준비)
                               │
                    CPU 할당   │   CPU 선점
                  ┌────────────┘────────────┐
                  ↓                         ↑
              RUNNING (실행) ──────────────→ READY
                  │
    I/O 요청,     │   I/O 완료,
    sleep()       ↓   신호 수신
              SLEEPING (대기)
                  │
              exit()
                  ↓
              ZOMBIE (좀비) → 부모 wait() → 소멸
```

### PID 1 - init/systemd

```bash
# PID 1 확인
ps -p 1
cat /proc/1/comm  # systemd

# 프로세스 트리 전체
pstree
pstree -p         # PID 포함
pstree -u         # 사용자 포함
```

## 6.2 프로세스 모니터링

### ps - 프로세스 스냅샷

```bash
# 현재 터미널의 프로세스
ps

# 전체 프로세스 (BSD 스타일)
ps aux

# 전체 프로세스 (System V 스타일)
ps -ef

# 특정 프로세스 찾기
ps aux | grep nginx
ps -ef | grep python

# 프로세스 트리
ps axjf
ps -ejH

# 특정 사용자의 프로세스
ps -u student

# 출력 형식 커스터마이징
ps -eo pid,ppid,user,%cpu,%mem,stat,cmd
```

```
ps aux 출력 해석:
USER   PID  %CPU %MEM    VSZ   RSS TTY  STAT START   TIME COMMAND
root     1   0.0  0.3 168548 12345 ?    Ss   10:00   0:01 /sbin/init
student 1234  2.1  1.5 123456 60000 pts/0 Sl  10:15   0:30 python3 app.py

STAT 코드:
R - Running (실행 중)
S - Sleeping (인터럽트 가능 대기)
D - Disk sleep (인터럽트 불가 대기, I/O 대기)
Z - Zombie
T - Stopped
< - 높은 우선순위
N - 낮은 우선순위
```

### top - 실시간 모니터링

```bash
# top 실행
top

# top 단축키:
# q    - 종료
# h    - 도움말
# M    - 메모리 사용량순 정렬
# P    - CPU 사용량순 정렬
# k    - 프로세스 종료 (PID 입력)
# r    - renice (우선순위 변경)
# u    - 특정 사용자 필터
# 1    - CPU 코어별 표시
# W    - 설정 저장
```

### htop - 향상된 top

```bash
# 설치
sudo apt install htop

# 실행
htop

# htop 단축키:
# F2      - 설정
# F3/F4   - 검색/필터
# F5      - 트리뷰
# F6      - 정렬 기준
# F9      - 프로세스 종료
# F10/q   - 종료
# Space   - 태그 (다중 선택)
```

## 6.3 프로세스 제어

### 포그라운드 / 백그라운드

```bash
# 백그라운드로 실행 (&)
python3 server.py &
sleep 100 &

# 작업 목록 확인
jobs
jobs -l  # PID 포함

# 백그라운드 작업을 포그라운드로
fg          # 가장 최근 작업
fg %1       # 1번 작업

# 포그라운드 작업을 백그라운드로
Ctrl+Z      # 일시 정지
bg          # 백그라운드에서 계속 실행
bg %1
```

### nohup - 터미널 종료 후에도 실행

```bash
# 터미널 종료 시 SIGHUP 무시
nohup python3 server.py &
nohup python3 server.py > output.log 2>&1 &

# 출력은 nohup.out 또는 지정 파일에 저장
tail -f nohup.out
```

### disown - 현재 쉘에서 분리

```bash
# 실행 중인 작업을 쉘에서 분리
python3 server.py &
disown %1

# 모든 작업 분리
disown -a
```

## 6.4 시그널과 kill

### 주요 시그널

| 번호 | 이름 | 설명 |
|------|------|------|
| 1 | SIGHUP | 터미널 연결 종료, 설정 재로드 |
| 2 | SIGINT | Ctrl+C, 인터럽트 |
| 9 | SIGKILL | 강제 종료 (무시 불가) |
| 15 | SIGTERM | 정상 종료 요청 (기본) |
| 18 | SIGCONT | 정지된 프로세스 재개 |
| 19 | SIGSTOP | 프로세스 정지 (무시 불가) |

```bash
# PID로 시그널 전송
kill 1234           # SIGTERM (기본)
kill -9 1234        # SIGKILL (강제)
kill -TERM 1234     # SIGTERM 명시
kill -HUP 1234      # 설정 재로드

# 프로세스 이름으로 종료
killall nginx
killall -9 zombie_process

# 패턴으로 종료
pkill -f "python3 app.py"
pkill -u student    # 특정 사용자의 모든 프로세스

# 프로세스 확인
pgrep nginx
pgrep -l python     # 이름 포함
```

### 좀비 프로세스 처리

```bash
# 좀비 프로세스 찾기
ps aux | grep Z

# 좀비의 부모 프로세스 찾기
ps -ef | grep Z
# PPID 확인 후 부모 프로세스에 SIGCHLD 전송
kill -CHLD <PPID>

# 또는 부모 프로세스 재시작
```

## 6.5 프로세스 우선순위

### nice와 renice

```bash
# nice 값 범위: -20 (최고 우선순위) ~ 19 (최저)
# 기본값: 0, 일반 사용자는 0~19만 설정 가능

# 낮은 우선순위로 실행
nice -n 10 python3 heavy_task.py

# 높은 우선순위 (root만 가능)
sudo nice -n -10 critical_process

# 실행 중인 프로세스 우선순위 변경
renice 15 -p 1234        # PID로
renice 10 -u student     # 사용자로
sudo renice -5 -p 1234   # root만 음수 가능
```

## 6.6 systemd 서비스 관리

### systemd 구조

```
systemd (PID 1)
├── 서비스 유닛 (.service)
├── 소켓 유닛 (.socket)
├── 타이머 유닛 (.timer)
├── 마운트 유닛 (.mount)
└── 타깃 유닛 (.target)  ← 런레벨 대체

유닛 파일 위치:
/lib/systemd/system/    ← 패키지 설치 시
/etc/systemd/system/    ← 관리자 설정 (우선순위 높음)
~/.config/systemd/user/ ← 사용자 서비스
```

### systemctl 기본 명령어

```bash
# 서비스 상태 확인
sudo systemctl status nginx
sudo systemctl status ssh

# 서비스 시작/중지/재시작
sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx
sudo systemctl reload nginx    # 설정만 재로드

# 부팅 시 자동 시작 설정
sudo systemctl enable nginx
sudo systemctl disable nginx

# 시작 + 활성화 한번에
sudo systemctl enable --now nginx

# 서비스 목록
systemctl list-units --type=service
systemctl list-units --type=service --state=running
systemctl list-units --type=service --state=failed
```

### 서비스 유닛 파일 작성

```bash
# 커스텀 서비스 예시
sudo nano /etc/systemd/system/myapp.service
```

```ini
[Unit]
Description=My Python Application
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
User=student
Group=student
WorkingDirectory=/opt/myapp
ExecStart=/usr/bin/python3 /opt/myapp/app.py
ExecReload=/bin/kill -HUP $MAINPID
Restart=always
RestartSec=3
StandardOutput=journal
StandardError=journal
Environment=FLASK_ENV=production
EnvironmentFile=/opt/myapp/.env

[Install]
WantedBy=multi-user.target
```

```bash
# 새 서비스 파일 인식
sudo systemctl daemon-reload

# 서비스 활성화 및 시작
sudo systemctl enable --now myapp

# 로그 확인
sudo journalctl -u myapp -f
sudo journalctl -u myapp --since "1 hour ago"
```

### systemd 타깃 (런레벨)

```bash
# 현재 타깃 확인
systemctl get-default

# 타깃 변경
sudo systemctl set-default multi-user.target   # CLI 모드
sudo systemctl set-default graphical.target    # GUI 모드

# 즉시 전환
sudo systemctl isolate rescue.target           # 복구 모드

# 주요 타깃
# poweroff.target   - 전원 끄기
# rescue.target     - 단일 사용자 모드
# multi-user.target - 다중 사용자 (CLI)
# graphical.target  - 다중 사용자 (GUI)
# reboot.target     - 재부팅
```

## 핵심 요약

- **프로세스**: PID, PPID, 상태(R/S/D/Z/T)로 관리
- **ps aux**: 전체 프로세스 스냅샷, **top/htop**: 실시간 모니터링
- **kill 시그널**: SIGTERM(15)=정상종료, SIGKILL(9)=강제종료
- **&, nohup, disown**: 백그라운드 실행 방법
- **nice/renice**: CPU 우선순위(-20~19)
- **systemctl**: 서비스 시작/중지/활성화/상태확인
- **systemd 유닛 파일**: /etc/systemd/system/에 .service 파일 작성
