## 8.1 SSH 프로토콜 이해

### SSH 연결 과정

```
클라이언트                        서버
    │                               │
    │──── TCP 연결 (포트 22) ───────→│
    │                               │
    │←── 서버 공개키 전송 ───────────│
    │                               │
    │  키 교환 (Diffie-Hellman)      │
    │←─────────────────────────────→│
    │                               │
    │  세션 키 생성 (대칭 암호화)     │
    │                               │
    │──── 사용자 인증 ──────────────→│
    │  (비밀번호 or 공개키)          │
    │                               │
    │←──── 암호화된 쉘 세션 ─────────│
```

### SSH 버전과 암호화

```bash
# SSH 버전 확인
ssh -V

# 지원하는 암호화 알고리즘 확인
ssh -Q cipher
ssh -Q mac
ssh -Q kex
ssh -Q key
```

## 8.2 SSH 기본 사용법

### 접속

```bash
# 기본 접속 (현재 사용자명으로)
ssh 192.168.1.100

# 사용자 지정
ssh student@192.168.1.100

# 포트 지정
ssh -p 2222 student@192.168.1.100

# 특정 키 파일 사용
ssh -i ~/.ssh/mykey.pem student@192.168.1.100

# 첫 접속 시 호스트 키 확인
# The authenticity of host '192.168.1.100' can't be established.
# ED25519 key fingerprint is SHA256:xxxx...
# Are you sure you want to continue connecting (yes/no)?
```

### SSH 설정 파일 (~/.ssh/config)

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
nano ~/.ssh/config
```

```
# ~/.ssh/config

# 기본 설정
Host *
    ServerAliveInterval 60
    ServerAliveCountMax 3
    AddKeysToAgent yes

# 개발 서버
Host dev
    HostName 192.168.1.100
    User student
    Port 22
    IdentityFile ~/.ssh/dev_key

# 프로덕션 서버
Host prod
    HostName 10.0.1.50
    User ubuntu
    Port 2222
    IdentityFile ~/.ssh/prod_key.pem
    StrictHostKeyChecking yes

# 배스천 호스트를 통한 터널링
Host internal
    HostName 10.0.2.10
    User admin
    ProxyJump bastion
```

```bash
# 설정 파일로 간단히 접속
ssh dev
ssh prod
```

## 8.3 키 기반 인증

### SSH 키 생성

```bash
# ED25519 키 생성 (최신, 권장)
ssh-keygen -t ed25519 -C "student@company.com"

# RSA 키 생성 (호환성)
ssh-keygen -t rsa -b 4096 -C "student@company.com"

# 키 파일 위치와 이름 지정
ssh-keygen -t ed25519 -f ~/.ssh/myserver_key

# 생성되는 파일:
# ~/.ssh/id_ed25519      (개인키 - 절대 공유 금지!)
# ~/.ssh/id_ed25519.pub  (공개키 - 서버에 등록)

# 권한 설정
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
```

### 서버에 공개키 등록

```bash
# 방법 1: ssh-copy-id (권장)
ssh-copy-id student@192.168.1.100
ssh-copy-id -i ~/.ssh/mykey.pub student@192.168.1.100

# 방법 2: 수동 등록
cat ~/.ssh/id_ed25519.pub | ssh student@192.168.1.100 \
  "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

# 방법 3: 서버에서 직접 편집
# 서버에서:
mkdir -p ~/.ssh
chmod 700 ~/.ssh
nano ~/.ssh/authorized_keys  # 공개키 내용 붙여넣기
chmod 600 ~/.ssh/authorized_keys
```

### SSH 에이전트

```bash
# SSH 에이전트 시작
eval $(ssh-agent)

# 키 추가
ssh-add ~/.ssh/id_ed25519
ssh-add ~/.ssh/myserver_key

# 등록된 키 확인
ssh-add -l

# 키 제거
ssh-add -d ~/.ssh/myserver_key

# ~/.bashrc에 자동 실행 추가
echo 'eval $(ssh-agent) && ssh-add ~/.ssh/id_ed25519' >> ~/.bashrc
```

## 8.4 SSH 서버 보안 설정

### sshd_config 보안 강화

```bash
sudo nano /etc/ssh/sshd_config
```

```ini
# /etc/ssh/sshd_config 보안 설정

# 포트 변경 (22 → 다른 포트)
Port 2222

# root 직접 로그인 금지
PermitRootLogin no

# 비밀번호 인증 비활성화 (키 설정 후)
PasswordAuthentication no
ChallengeResponseAuthentication no

# 공개키 인증 활성화
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys

# 빈 비밀번호 금지
PermitEmptyPasswords no

# X11 포워딩 (필요 시만)
X11Forwarding no

# 최대 인증 시도 횟수
MaxAuthTries 3

# 최대 세션 수
MaxSessions 5

# 로그인 허용 사용자
AllowUsers student admin

# 연결 타임아웃
LoginGraceTime 30
ClientAliveInterval 300
ClientAliveCountMax 2

# 프로토콜 버전
Protocol 2

# 강력한 암호화만 허용
Ciphers chacha20-poly1305@openssh.com,aes256-gcm@openssh.com
MACs hmac-sha2-512-etm@openssh.com,hmac-sha2-256-etm@openssh.com
KexAlgorithms curve25519-sha256,diffie-hellman-group-exchange-sha256
```

```bash
# 설정 문법 검사
sudo sshd -t

# 적용
sudo systemctl restart ssh
```

## 8.5 SCP와 rsync 파일 전송

### SCP - 보안 복사

```bash
# 로컬 → 원격
scp file.txt student@192.168.1.100:/home/student/
scp -r directory/ student@192.168.1.100:/tmp/

# 원격 → 로컬
scp student@192.168.1.100:/var/log/nginx/error.log ./

# 원격 → 원격
scp server1:/file.txt server2:/destination/

# 포트 지정
scp -P 2222 file.txt student@192.168.1.100:/tmp/

# 진행 상황 표시
scp -v file.txt student@server:/tmp/
```

### rsync - 효율적인 동기화

```bash
# 기본 동기화 (로컬)
rsync -av source/ destination/

# 원격 동기화
rsync -avz ./local/ student@server:/remote/

# 삭제 동기화 (소스에 없는 파일 제거)
rsync -avz --delete ./local/ student@server:/remote/

# 특정 파일 제외
rsync -avz --exclude='*.log' --exclude='.git' ./local/ student@server:/remote/

# 드라이런 (실제 실행 없이 확인)
rsync -avzn ./local/ student@server:/remote/

# 압축 + 체크섬 검증
rsync -avzc ./local/ student@server:/remote/

# SSH 포트 지정
rsync -avz -e "ssh -p 2222" ./local/ student@server:/remote/

# 백업 예시 (날짜별)
rsync -avz --backup --backup-dir=../backup-$(date +%Y%m%d) \
  ./data/ student@server:/backup/
```

## 8.6 SSH 포트 포워딩 (터널링)

### 로컬 포트 포워딩

```bash
# 원격 서버의 포트를 로컬로 포워딩
# ssh -L [로컬포트]:[원격호스트]:[원격포트] [SSH서버]

# 원격 MySQL(3306)을 로컬 3307로
ssh -L 3307:localhost:3306 student@server
# 이후 로컬에서: mysql -h 127.0.0.1 -P 3307

# 내부망 웹서버를 로컬로 접속
ssh -L 8080:internal-web:80 student@bastion
# 이후 로컬에서: http://localhost:8080
```

### 원격 포트 포워딩

```bash
# 로컬 서비스를 원격 서버를 통해 공개
# ssh -R [원격포트]:[로컬호스트]:[로컬포트] [SSH서버]

# 로컬 3000번을 원격 서버 8080으로 공개
ssh -R 8080:localhost:3000 student@server
```

### 동적 포트 포워딩 (SOCKS 프록시)

```bash
# SOCKS5 프록시 생성
ssh -D 1080 student@server

# 브라우저에서 SOCKS 프록시 설정: 127.0.0.1:1080
```

### 백그라운드 터널

```bash
# 백그라운드로 실행 (-N: 명령 없음, -f: 백그라운드)
ssh -NfL 3307:localhost:3306 student@server

# 터널 종료
ps aux | grep ssh
kill <PID>
```

## 8.7 실전: 멀티홉 SSH (Jump Host)

```bash
# 배스천 → 내부 서버 접속
ssh -J student@bastion.example.com admin@10.0.1.50

# ~/.ssh/config 설정
Host internal
    HostName 10.0.1.50
    User admin
    ProxyJump student@bastion.example.com

# 간단히 접속
ssh internal
```

## 핵심 요약

- **키 기반 인증**: ssh-keygen → ssh-copy-id → PasswordAuthentication no
- **~/.ssh/config**: Host 별칭으로 복잡한 설정 단순화
- **scp**: 단순 파일 전송, **rsync**: 대용량/동기화에 적합
- **로컬 포트 포워딩**: `-L` 옵션으로 원격 서비스를 로컬에서 접속
- **보안 설정**: root 로그인 금지, 비밀번호 인증 비활성화, 포트 변경
- **fail2ban**: SSH 브루트포스 방어 필수 (챕터 9에서 다룸)
