## 1.1 리눅스의 탄생과 역사

### UNIX에서 Linux로

1969년 AT&T 벨 연구소에서 Ken Thompson과 Dennis Ritchie가 UNIX를 개발했습니다. UNIX는 멀티태스킹과 멀티유저를 지원하는 혁신적인 운영체제였지만, 상업적 라이선스로 인해 사용이 제한적이었습니다.

```
UNIX 계보:
UNIX (1969, AT&T)
├── BSD (1977, UC Berkeley)
│   ├── FreeBSD
│   ├── NetBSD
│   └── macOS (Darwin)
├── System V
│   └── AIX, HP-UX, Solaris
└── GNU/Linux (1991, Linus Torvalds)
    ├── Debian → Ubuntu, Mint
    ├── Red Hat → Fedora, CentOS, RHEL
    └── Arch Linux → Manjaro
```

### Linus Torvalds와 Linux 커널

1991년 핀란드 헬싱키 대학교 학생이었던 **Linus Torvalds**는 개인 프로젝트로 운영체제 커널을 개발하기 시작했습니다. 그의 유명한 메시지:

> "저는 386(486) AT 클론을 위한 자유로운(무료) 운영체제를 만들고 있습니다. 단지 취미로, GNU처럼 크고 전문적이지는 않을 것입니다."

Linux는 오픈소스로 공개되어 전 세계 개발자들의 기여를 받으며 빠르게 성장했습니다.

### GNU 프로젝트와의 만남

Richard Stallman이 1983년 시작한 **GNU 프로젝트**는 자유 소프트웨어 도구들(gcc, bash, glibc 등)을 제공했지만 커널이 없었습니다. Linux 커널과 GNU 도구들이 결합되어 완전한 운영체제인 **GNU/Linux**가 탄생했습니다.

## 1.2 리눅스 커널 구조

### 운영체제의 계층 구조

```
┌─────────────────────────────────┐
│         사용자 애플리케이션       │  ← 웹 브라우저, 편집기, 게임
├─────────────────────────────────┤
│         시스템 라이브러리         │  ← glibc, libstdc++
├─────────────────────────────────┤
│         시스템 콜 인터페이스      │  ← read(), write(), fork()
├─────────────────────────────────┤
│                                 │
│         리눅스 커널              │  ← 핵심 운영체제
│  ┌──────┬──────┬──────┬──────┐  │
│  │프로세스│메모리│파일  │네트워크│  │
│  │관리   │관리  │시스템│스택   │  │
│  └──────┴──────┴──────┴──────┘  │
├─────────────────────────────────┤
│         하드웨어 드라이버         │  ← CPU, 메모리, 디스크, NIC
├─────────────────────────────────┤
│         하드웨어                 │  ← 물리적 장치
└─────────────────────────────────┘
```

### 커널의 주요 기능

| 기능 | 설명 |
|------|------|
| 프로세스 관리 | CPU 스케줄링, 프로세스 생성/종료 |
| 메모리 관리 | 가상 메모리, 페이지 교체, 메모리 할당 |
| 파일 시스템 | ext4, xfs, btrfs 등 다양한 파일시스템 지원 |
| 네트워크 스택 | TCP/IP 프로토콜 구현 |
| 디바이스 드라이버 | 하드웨어 추상화 계층 제공 |
| 보안 | DAC, MAC, SELinux, AppArmor |

### 모놀리식 커널 vs 마이크로커널

Linux는 **모놀리식 커널(Monolithic Kernel)** 방식을 채택합니다.

```
모놀리식 커널 (Linux):           마이크로커널 (MINIX, QNX):
┌──────────────────┐             ┌──────────────────┐
│  단일 커널 공간   │             │   최소한의 커널   │
│ ┌──┬──┬──┬──┐   │             ├──────────────────┤
│ │FS│MM│NS│DD│   │             │  파일시스템 서버  │
│ └──┴──┴──┴──┘   │             │  드라이버 서버    │
└──────────────────┘             │  메모리 서버      │
장점: 성능 우수                   └──────────────────┘
단점: 버그 시 시스템 전체 영향     장점: 안정성
                                  단점: 성능 오버헤드
```

## 1.3 우분투(Ubuntu)란?

### 우분투의 탄생

**우분투(Ubuntu)**는 2004년 South African 기업가 **Mark Shuttleworth**가 설립한 **Canonical Ltd.**에서 만든 리눅스 배포판입니다. Debian을 기반으로 하며, 사용 편의성에 초점을 맞췄습니다.

이름 "Ubuntu"는 아프리카 반투족 철학에서 유래:
> **"I am what I am because of who we all are"**
> (우리 모두가 있기에 내가 있다)

### 우분투 버전 체계

우분투는 **6개월마다** 새 버전을 출시하며, **2년마다 LTS(Long Term Support)** 버전을 출시합니다.

```
버전 번호 형식: YY.MM
예: 24.04 LTS = 2024년 4월, 장기 지원

지원 기간:
일반 버전:  9개월
LTS 버전:  5년 (서버는 10년 ESM)

주요 LTS 버전:
- 20.04 LTS (Focal Fossa)     ← 2025년까지 지원
- 22.04 LTS (Jammy Jellyfish) ← 2027년까지 지원
- 24.04 LTS (Noble Numbat)    ← 2029년까지 지원
```

### 우분투 에디션

| 에디션 | 대상 | 특징 |
|--------|------|------|
| Ubuntu Desktop | 일반 사용자 | GNOME 데스크톱 환경 |
| Ubuntu Server | 서버 관리자 | GUI 없음, 최소 설치 |
| Ubuntu Core | IoT 기기 | 스냅 패키지 기반 |
| Kubuntu | KDE 선호자 | KDE Plasma 환경 |
| Xubuntu | 저사양 PC | XFCE 환경 |

## 1.5 리눅스를 배워야 하는 이유

### 서버 시장 점유율

```
전 세계 서버 운영체제 시장 (2024):
Linux    ████████████████████████ 96.3%
Windows  ██ 1.9%
기타     ▌ 1.8%

클라우드 VM 사용 비율:
AWS EC2   Linux 기반 인스턴스: 92%
Azure     Linux VM: 60% 이상
GCP       Linux 기반: 85%
```

### 취업 시장에서의 가치

- **DevOps / 클라우드 엔지니어**: Linux 필수 역량
- **백엔드 개발자**: 서버 배포/운영에 필수
- **데이터 엔지니어**: Hadoop, Spark 등 빅데이터 플랫폼
- **보안 전문가**: 침투 테스트 도구 대부분 Linux 기반
- **임베디드 개발자**: Android, IoT 기기 모두 Linux 커널

### 개발자 도구 생태계

```bash
# 현대 개발에 필수적인 Linux 기반 도구들

Kubernetes  # 컨테이너 오케스트레이션
Git         # 버전 관리
Nginx       # 웹 서버
MySQL/PostgreSQL  # 데이터베이스
Python/Node.js    # 서버사이드 개발
```

## 1.6 오픈소스 라이선스 이해

### 주요 오픈소스 라이선스

| 라이선스 | 특징 | 대표 소프트웨어 |
|----------|------|----------------|
| GPL v2/v3 | 수정 시 동일 라이선스 적용 | Linux 커널, GCC |
| LGPL | 라이브러리 링크는 자유 | glibc |
| MIT | 가장 자유로운 라이선스 | X11, jQuery |
| Apache 2.0 | 특허권 명시 | Kubernetes, Android |
| BSD | MIT와 유사, 광고 조항 | FreeBSD, PostgreSQL |

### GPL의 중요성

Linux 커널은 **GPL v2** 라이선스입니다. 이는:
- 소스코드를 자유롭게 사용, 수정, 배포 가능
- 수정한 코드도 GPL로 공개해야 함 (카피레프트)
- 상업적 사용 가능하나 소스 공개 의무

## 핵심 요약

- **Linux**는 1991년 Linus Torvalds가 개발한 오픈소스 운영체제 커널
- **GNU/Linux**는 GNU 도구 + Linux 커널의 조합
- **Ubuntu**는 Debian 기반의 사용자 친화적 배포판, 6개월 주기 출시
- **LTS 버전**은 5년간 장기 지원, 서버 환경에 권장
- 전 세계 서버의 96% 이상이 Linux 기반으로 운영
- 현대 개발자에게 Linux는 선택이 아닌 필수 역량
