# 챕터 10: APT 패키지 관리

> 우분투의 패키지 관리 시스템을 완전히 마스터한다

## 학습 목표
- APT 패키지 관리 시스템의 구조를 이해한다
- apt 명령어로 소프트웨어를 설치/관리할 수 있다
- PPA와 서드파티 저장소를 추가할 수 있다
- 소스에서 직접 컴파일하여 설치할 수 있다

## 10.1 패키지 관리 시스템 이해

### Debian 패키지 시스템 구조

```
사용자
  ↓
apt (고수준 인터페이스)
  ↓
apt-get / apt-cache (중간 레이어)
  ↓
dpkg (저수준 패키지 관리)
  ↓
.deb 패키지 파일

저장소(Repository):
Ubuntu Server ──→ /etc/apt/sources.list
PPA ──────────→ /etc/apt/sources.list.d/
서드파티 ─────→ /etc/apt/sources.list.d/
```

### 저장소 구성

```bash
# 기본 저장소 확인
cat /etc/apt/sources.list
```

```
deb http://archive.ubuntu.com/ubuntu jammy main restricted universe multiverse
deb http://archive.ubuntu.com/ubuntu jammy-updates main restricted universe multiverse
deb http://security.ubuntu.com/ubuntu jammy-security main restricted universe multiverse

컴포넌트:
main        - 공식 지원, 오픈소스
restricted  - 공식 지원, 비오픈소스 드라이버
universe    - 커뮤니티 지원
multiverse  - 비자유 소프트웨어
```

## 10.2 apt 명령어

### 패키지 목록 업데이트

```bash
# 저장소 패키지 목록 업데이트 (설치는 하지 않음)
sudo apt update

# 업그레이드 가능한 패키지 목록
apt list --upgradable
```

### 패키지 설치와 업그레이드

```bash
# 패키지 설치
sudo apt install nginx
sudo apt install nginx mysql-server php-fpm  # 여러 패키지

# 특정 버전 설치
sudo apt install nginx=1.18.0-0ubuntu1

# 설치 없이 다운로드만
sudo apt install -d nginx

# 확인 없이 설치 (-y)
sudo apt install -y nginx

# 시뮬레이션 (실제 설치 없이 확인)
sudo apt install --simulate nginx

# 업그레이드
sudo apt upgrade              # 기존 패키지 업그레이드
sudo apt full-upgrade         # 의존성 변경 포함 업그레이드
sudo apt dist-upgrade         # full-upgrade와 동일
```

### 패키지 제거

```bash
# 패키지 제거 (설정 파일 유지)
sudo apt remove nginx

# 패키지 + 설정 파일 완전 제거
sudo apt purge nginx

# 의존성으로만 설치된 불필요 패키지 제거
sudo apt autoremove

# 한 번에 완전 제거
sudo apt purge --autoremove nginx

# 로컬 캐시 정리
sudo apt clean          # 다운로드된 패키지 파일 삭제
sudo apt autoclean      # 오래된 버전만 삭제
```

### 패키지 검색과 정보

```bash
# 패키지 검색
apt search nginx
apt search "web server"

# 패키지 정보 확인
apt show nginx
apt-cache show nginx

# 패키지 의존성 확인
apt depends nginx
apt rdepends nginx      # 역의존성 (누가 이 패키지를 필요로 하나)

# 설치된 패키지 목록
apt list --installed
apt list --installed | grep nginx
dpkg -l
dpkg -l | grep nginx

# 파일이 어느 패키지에 속하는지
dpkg -S /usr/bin/nginx

# 패키지가 설치할 파일 목록
dpkg -L nginx
```

## 10.3 dpkg - 저수준 패키지 관리

```bash
# .deb 파일 설치
sudo dpkg -i package.deb

# 의존성 문제 해결 후 재시도
sudo apt install -f

# 패키지 제거
sudo dpkg -r package-name
sudo dpkg -P package-name    # 설정 파일 포함 제거

# 설치된 패키지 정보
dpkg -s package-name

# 패키지 파일 추출 (설치 없이)
dpkg -x package.deb /tmp/extract/

# 패키지 내용 확인
dpkg -c package.deb
```

## 10.4 PPA 추가

### PPA(Personal Package Archive) 란?

PPA는 개인이나 팀이 운영하는 우분투 패키지 저장소입니다. 공식 저장소보다 최신 버전이나 추가 소프트웨어를 제공합니다.

```bash
# PPA 추가
sudo add-apt-repository ppa:deadsnakes/python
sudo add-apt-repository ppa:ondrej/php

# 목록 업데이트 후 설치
sudo apt update
sudo apt install python3.12

# PPA 제거
sudo add-apt-repository --remove ppa:deadsnakes/python
# 또는
sudo ppa-purge ppa:deadsnakes/python
```

### 서드파티 저장소 추가

```bash
# 예: Docker 공식 저장소
# 1. GPG 키 추가
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 2. 저장소 추가
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list

# 3. 업데이트 및 설치
sudo apt update
sudo apt install docker-ce

# 예: Node.js 공식 저장소
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

## 10.5 소스 컴파일 설치

### 빌드 환경 준비

```bash
# 빌드 필수 도구
sudo apt install build-essential
sudo apt install autoconf automake libtool

# 개발 라이브러리
sudo apt install libssl-dev libpcre3-dev zlib1g-dev
```

### 소스에서 컴파일 (예: Nginx)

```bash
# 소스 다운로드
wget http://nginx.org/download/nginx-1.25.0.tar.gz
tar -xzf nginx-1.25.0.tar.gz
cd nginx-1.25.0

# 의존성 설치
sudo apt install -y libpcre3 libpcre3-dev zlib1g zlib1g-dev \
  libssl-dev libgd-dev

# 설정 (configure)
./configure \
  --prefix=/opt/nginx \
  --with-http_ssl_module \
  --with-http_v2_module \
  --with-http_gzip_static_module

# 컴파일
make -j$(nproc)     # CPU 코어 수만큼 병렬 컴파일

# 설치
sudo make install

# 확인
/opt/nginx/sbin/nginx -v
```

### checkinstall - 소스 설치를 패키지로 변환

```bash
sudo apt install checkinstall

# make install 대신 checkinstall 사용
sudo checkinstall make install
# .deb 패키지 생성 및 설치
# dpkg로 관리 가능
```

## 10.6 패키지 보류와 우선순위

### 패키지 버전 고정 (Hold)

```bash
# 패키지 업그레이드 방지
sudo apt-mark hold nginx
sudo apt-mark hold linux-image-generic

# 보류 목록 확인
apt-mark showhold

# 보류 해제
sudo apt-mark unhold nginx
```

### apt preferences - 저장소 우선순위

```bash
# /etc/apt/preferences.d/에 파일 생성
sudo nano /etc/apt/preferences.d/nginx-priority
```

```
Package: nginx
Pin: version 1.18.*
Pin-Priority: 1001
```

## 핵심 요약

| 명령어 | 용도 |
|--------|------|
| `apt update` | 패키지 목록 갱신 |
| `apt upgrade` | 패키지 업그레이드 |
| `apt install pkg` | 패키지 설치 |
| `apt remove pkg` | 패키지 제거 |
| `apt purge pkg` | 설정 포함 완전 제거 |
| `apt search` | 패키지 검색 |
| `apt show` | 패키지 정보 |
| `dpkg -l` | 설치 목록 |
| `dpkg -S file` | 파일의 패키지 확인 |

- **apt update는 설치가 아님**: 목록만 갱신
- **PPA**: 최신 버전이 필요할 때, 신뢰할 수 있는 출처만
- **소스 설치**: 커스텀 옵션이 필요할 때만 사용
- **apt-mark hold**: 특정 버전 유지 필요 시
