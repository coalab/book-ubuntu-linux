## 5.1 파일 권한 이해

### 권한 표기 방식

```
ls -l 출력:
-rwxr-xr--  1  student  devteam  4096  Jan 15  file.sh
│└─┤└─┤└─┤     │        │
│  │  │  │     │        └── 그룹명
│  │  │  │     └─────────── 소유자
│  │  │  └───────────────── other 권한 (r--)
│  │  └──────────────────── group 권한 (r-x)
│  └─────────────────────── owner 권한 (rwx)
└────────────────────────── 파일 유형 (- = 일반파일)

파일 유형:
- : 일반 파일
d : 디렉토리
l : 심볼릭 링크
c : 문자 장치
b : 블록 장치
p : 파이프
s : 소켓
```

### 권한의 의미

| 권한 | 파일 | 디렉토리 |
|------|------|---------|
| r (4) | 읽기 가능 | 목록 조회 (ls) |
| w (2) | 쓰기/수정 가능 | 파일 생성/삭제 |
| x (1) | 실행 가능 | 진입 가능 (cd) |

```
8진수 표기:
rwx = 4+2+1 = 7
rw- = 4+2+0 = 6
r-x = 4+0+1 = 5
r-- = 4+0+0 = 4
--- = 0+0+0 = 0

예: rwxr-xr-- = 754
     7   5   4
```

## 5.2 chmod - 권한 변경

### 8진수 방식

```bash
# rwxr-xr-- (소유자:모두, 그룹:읽기+실행, 기타:읽기만)
chmod 754 file.txt

# 일반적인 권한 설정
chmod 644 document.txt    # rw-r--r-- (일반 파일)
chmod 755 script.sh       # rwxr-xr-x (실행 파일)
chmod 700 private.key     # rwx------ (개인 파일)
chmod 777 public.txt      # rwxrwxrwx (모두 허용, 위험!)

# 재귀적으로 적용
chmod -R 755 /var/www/html
```

### 심볼릭 방식

```bash
# u=소유자, g=그룹, o=기타, a=모두
# +=추가, -=제거, ==설정

# 실행 권한 추가
chmod +x script.sh
chmod u+x script.sh       # 소유자에만

# 쓰기 권한 제거
chmod go-w sensitive.txt  # 그룹과 기타에서 쓰기 제거

# 정확하게 설정
chmod u=rw,go=r file.txt  # 소유자:rw, 나머지:r

# 재귀적으로
chmod -R u=rwX,go=rX /home/shared
# X = 디렉토리이거나 이미 실행 권한이 있을 때만 x 설정
```

## 5.3 chown, chgrp - 소유권 변경

```bash
# 소유자 변경 (root 권한 필요)
sudo chown student file.txt

# 소유자와 그룹 함께 변경
sudo chown student:devteam file.txt

# 그룹만 변경
sudo chgrp devteam file.txt
# 또는
sudo chown :devteam file.txt

# 재귀적 변경
sudo chown -R www-data:www-data /var/www/html

# 심볼릭 링크가 아닌 원본 파일 변경
sudo chown -h student symlink
```

## 5.4 특수 권한 비트

### SetUID, SetGID, Sticky Bit

```bash
# SetUID (4000): 실행 시 파일 소유자 권한으로 실행
# 예: /usr/bin/passwd (일반 사용자가 root 권한 파일 수정)
chmod u+s file
chmod 4755 file
ls -l file  # -rwsr-xr-x

# SetGID (2000): 실행 시 그룹 권한으로 실행
# 디렉토리에 설정 시 하위 파일이 같은 그룹 상속
chmod g+s directory
chmod 2755 directory
ls -l  # drwxr-sr-x

# Sticky Bit (1000): 디렉토리에서 자신의 파일만 삭제 가능
# 예: /tmp 디렉토리
chmod +t /shared
chmod 1777 /tmp
ls -l  # drwxrwxrwt
```

```bash
# SetUID 설정된 파일 찾기 (보안 점검)
find / -perm /4000 -type f 2>/dev/null
find / -perm /2000 -type f 2>/dev/null
```

### umask - 기본 권한 마스크

```bash
# 현재 umask 확인
umask
# 0022

# umask 계산:
# 파일 기본 권한: 666 - 022 = 644 (rw-r--r--)
# 디렉토리 기본 권한: 777 - 022 = 755 (rwxr-xr-x)

# umask 변경 (현재 세션)
umask 027   # 파일: 640, 디렉토리: 750

# 영구 설정 (~/.bashrc)
echo "umask 027" >> ~/.bashrc
```

## 5.5 사용자 계정 관리

### 사용자 정보 파일

```bash
# /etc/passwd 구조
cat /etc/passwd
# username:x:UID:GID:comment:home:shell
# student:x:1000:1000:Student User:/home/student:/bin/bash

# /etc/shadow 구조 (root만 읽기 가능)
sudo cat /etc/shadow
# username:암호화된패스워드:최종변경일:최소:최대:경고:비활성:만료:예약

# /etc/group 구조
cat /etc/group
# groupname:x:GID:members
# devteam:x:1001:student,bob
```

### 사용자 추가

```bash
# 기본 사용자 추가 (홈 디렉토리 자동 생성)
sudo useradd -m student

# 상세 옵션으로 추가
sudo useradd \
  -m \                    # 홈 디렉토리 생성
  -s /bin/bash \          # 기본 쉘
  -c "Kim Student" \      # 코멘트 (이름)
  -G sudo,docker \        # 추가 그룹
  student

# 비밀번호 설정
sudo passwd student

# adduser (대화형, 더 친화적)
sudo adduser newuser
```

### 사용자 수정

```bash
# 쉘 변경
sudo usermod -s /bin/zsh student

# 추가 그룹에 포함
sudo usermod -aG docker student
sudo usermod -aG sudo student

# 홈 디렉토리 변경
sudo usermod -d /new/home -m student

# 계정 잠금
sudo usermod -L student
sudo passwd -l student

# 계정 잠금 해제
sudo usermod -U student
sudo passwd -u student

# 계정 만료 설정
sudo usermod -e 2024-12-31 student
```

### 사용자 삭제

```bash
# 사용자 삭제 (홈 디렉토리 유지)
sudo userdel student

# 홈 디렉토리와 메일 스풀도 삭제
sudo userdel -r student

# 삭제 전 소유 파일 확인
find / -user student -type f 2>/dev/null
```

## 5.6 그룹 관리

```bash
# 그룹 생성
sudo groupadd devteam
sudo groupadd -g 2000 devteam  # GID 지정

# 그룹 수정
sudo groupmod -n newname devteam   # 이름 변경
sudo groupmod -g 2001 devteam      # GID 변경

# 그룹 삭제
sudo groupdel devteam

# 사용자의 그룹 확인
groups student
id student

# 현재 사용자 그룹 정보
groups
```

## 5.7 sudo 설정

### /etc/sudoers 파일

```bash
# visudo로 안전하게 편집 (문법 검사)
sudo visudo

# /etc/sudoers.d/에 별도 파일 생성 (권장)
sudo visudo -f /etc/sudoers.d/student
```

```
# /etc/sudoers 주요 구문

# 사용자 권한
student ALL=(ALL:ALL) ALL       # 모든 권한
student ALL=(ALL) NOPASSWD: ALL # 비밀번호 없이

# 특정 명령어만 허용
student ALL=(ALL) /usr/bin/apt, /usr/bin/systemctl

# 그룹 권한
%sudo   ALL=(ALL:ALL) ALL       # sudo 그룹 전체
%devteam ALL=(ALL) /usr/bin/docker

# 별칭 정의
Cmnd_Alias WEBOPS = /usr/bin/systemctl restart nginx, \
                    /usr/bin/systemctl reload nginx
student ALL=(ALL) WEBOPS
```

```bash
# sudo 사용 예
sudo apt update
sudo -u www-data ls /var/www   # 다른 사용자로 실행
sudo -i                        # root 쉘 (환경변수 포함)
sudo -s                        # root 쉘 (현재 환경변수)

# sudo 로그 확인
sudo grep sudo /var/log/auth.log
```

## 5.8 PAM과 계정 보안

### 비밀번호 정책 설정

```bash
# PAM 모듈 설치
sudo apt install libpam-pwquality

# 비밀번호 정책 설정
sudo nano /etc/security/pwquality.conf
```

```ini
# /etc/security/pwquality.conf
minlen = 12          # 최소 길이
minclass = 3         # 최소 문자 클래스 수 (대/소/숫자/특수)
maxrepeat = 3        # 같은 문자 연속 최대 횟수
dcredit = -1         # 숫자 최소 1개 필요
ucredit = -1         # 대문자 최소 1개 필요
lcredit = -1         # 소문자 최소 1개 필요
ocredit = -1         # 특수문자 최소 1개 필요
```

### 계정 잠금 정책

```bash
# faillock 설정 (로그인 실패 제한)
sudo nano /etc/security/faillock.conf
```

```ini
deny = 5             # 5회 실패 시 잠금
unlock_time = 300    # 300초 후 자동 해제
fail_interval = 900  # 900초 내 실패 횟수 누적
```

```bash
# 잠긴 계정 확인 및 해제
sudo faillock --user student
sudo faillock --user student --reset
```

## 핵심 요약

- **권한 3요소**: 읽기(r=4), 쓰기(w=2), 실행(x=1)
- **chmod**: 8진수(644, 755) 또는 심볼릭(u+x) 방식
- **chown**: 소유자 변경, **chgrp**: 그룹 변경
- **특수 권한**: SetUID(4), SetGID(2), Sticky(1)
- **useradd/usermod/userdel**: 사용자 계정 생명주기 관리
- **visudo**: sudoers 파일 안전하게 편집
- **최소 권한 원칙**: 필요한 권한만 최소한으로 부여
