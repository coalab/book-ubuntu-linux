# 챕터 15: cron과 자동화

> 반복 작업을 자동화하여 시스템 운영 효율을 높인다

## 학습 목표
- crontab 문법을 완전히 이해한다
- 다양한 자동화 작업을 설정할 수 있다
- systemd timer를 cron 대신 활용할 수 있다
- 로그 관리와 logrotate를 설정할 수 있다

---

## 15.1 cron 이해

### cron 서비스

```bash
# cron 서비스 상태 확인
sudo systemctl status cron

# 시작 및 활성화
sudo systemctl enable --now cron
```

### crontab 문법

```
┌──────── 분 (0-59)
│ ┌────── 시 (0-23)
│ │ ┌──── 일 (1-31)
│ │ │ ┌── 월 (1-12)
│ │ │ │ ┌ 요일 (0-7, 0과 7=일요일)
│ │ │ │ │
* * * * *  명령어

특수 문자:
*  = 모든 값
,  = 목록 (1,3,5)
-  = 범위 (1-5)
/  = 간격 (*/5 = 5분마다)
```

### 주요 cron 예제

```
# 매분 실행
* * * * * /opt/scripts/check.sh

# 매시간 30분에 실행
30 * * * * /opt/scripts/hourly.sh

# 매일 새벽 2시에 실행
0 2 * * * /opt/scripts/backup.sh

# 월요일~금요일 오전 9시에 실행
0 9 * * 1-5 /opt/scripts/weekday_report.sh

# 5분마다 실행
*/5 * * * * /opt/scripts/monitor.sh

# 매월 1일 오전 6시에 실행
0 6 1 * * /opt/scripts/monthly_report.sh

# 매주 일요일 새벽 3시에 실행
0 3 * * 0 /opt/scripts/weekly_cleanup.sh

# 분기별 (1,4,7,10월 1일)
0 0 1 1,4,7,10 * /opt/scripts/quarterly.sh
```

### crontab 관리

```bash
# 현재 사용자 crontab 편집
crontab -e

# crontab 목록 확인
crontab -l

# crontab 삭제
crontab -r

# 특정 사용자 crontab 편집 (root만 가능)
sudo crontab -u www-data -e
sudo crontab -u student -l

# 시스템 crontab (루트 작업)
sudo nano /etc/crontab
# 형식: 분 시 일 월 요일 사용자 명령어
# 0 2 * * * root /opt/scripts/backup.sh
```

## 15.3 실전 cron 스크립트

### 데이터베이스 백업

```bash
#!/bin/bash
# /opt/scripts/db_backup.sh

set -euo pipefail

BACKUP_DIR="/var/backups/mysql"
RETENTION=7
DATE=$(date +%Y%m%d_%H%M%S)
LOGFILE="/var/log/db_backup.log"

log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*" >> "$LOGFILE"; }

mkdir -p "$BACKUP_DIR"
log "백업 시작"

for DB in $(mysql -u root -p"$MYSQL_ROOT_PASSWORD" \
           -e "SHOW DATABASES;" 2>/dev/null | \
           grep -Ev "information_schema|performance_schema|mysql|sys|Database"); do
    log "백업 중: $DB"
    mysqldump -u root -p"$MYSQL_ROOT_PASSWORD" \
              --single-transaction "$DB" | \
              gzip > "$BACKUP_DIR/${DB}_${DATE}.sql.gz"
done

find "$BACKUP_DIR" -name "*.sql.gz" -mtime +"$RETENTION" -delete
log "백업 완료. 오래된 백업 삭제됨 (${RETENTION}일 초과)"
```

```bash
# crontab 등록
echo "0 2 * * * MYSQL_ROOT_PASSWORD=secret /opt/scripts/db_backup.sh" | \
  sudo crontab -
```

### 디스크 모니터링 및 알림

```bash
#!/bin/bash
# /opt/scripts/disk_alert.sh

THRESHOLD=85
EMAIL="admin@example.com"

df -h | grep -vE '^Filesystem|tmpfs|cdrom' | \
while read -r line; do
    usage=$(echo "$line" | awk '{print $5}' | sed 's/%//')
    partition=$(echo "$line" | awk '{print $6}')
    if [ "${usage:-0}" -ge $THRESHOLD ]; then
        subject="[경고] 디스크 사용률 ${usage}% - $partition"
        body="서버: $(hostname)\n파티션: $partition\n사용률: ${usage}%\n시간: $(date)"
        echo -e "$body" | mail -s "$subject" "$EMAIL"
        echo "경고 발송: $partition ${usage}%"
    fi
done
```

### 로그 수집 및 분석

```bash
#!/bin/bash
# /opt/scripts/log_report.sh

REPORT_FILE="/tmp/daily_report_$(date +%Y%m%d).txt"

{
echo "=== 일일 보고서 $(date) ==="
echo ""
echo "--- SSH 로그인 시도 ---"
grep "Failed password" /var/log/auth.log | wc -l | xargs -I{} echo "실패: {} 회"
grep "Accepted" /var/log/auth.log | wc -l | xargs -I{} echo "성공: {} 회"
echo ""
echo "--- Nginx 오류 ---"
tail -100 /var/log/nginx/error.log | grep "error" | wc -l | \
  xargs -I{} echo "오류 수: {} 개"
echo ""
echo "--- 시스템 상태 ---"
uptime
free -h
df -h /
} > "$REPORT_FILE"

mail -s "일일 서버 보고서 $(date +%Y-%m-%d)" admin@example.com < "$REPORT_FILE"
rm -f "$REPORT_FILE"
```

## 15.5 systemd Timer

### cron과 비교

| 특성 | cron | systemd timer |
|------|------|---------------|
| 로그 | 별도 설정 | journald 자동 |
| 의존성 | 없음 | 서비스 의존성 |
| 실패 처리 | 제한적 | 재시도 가능 |
| 정확도 | 분 단위 | 마이크로초 |

### systemd timer 설정

```bash
# 서비스 유닛 파일
sudo nano /etc/systemd/system/backup.service
```

```ini
[Unit]
Description=Database Backup Service
After=mysql.service

[Service]
Type=oneshot
ExecStart=/opt/scripts/db_backup.sh
User=backup
StandardOutput=journal
StandardError=journal
```

```bash
# 타이머 유닛 파일
sudo nano /etc/systemd/system/backup.timer
```

```ini
[Unit]
Description=Daily Backup Timer
Requires=backup.service

[Timer]
OnCalendar=*-*-* 02:00:00    # 매일 새벽 2시
RandomizedDelaySec=300        # 최대 5분 랜덤 지연
Persistent=true               # 놓친 실행 보완

[Install]
WantedBy=timers.target
```

```bash
# 활성화 및 시작
sudo systemctl daemon-reload
sudo systemctl enable --now backup.timer

# 타이머 목록 확인
systemctl list-timers
systemctl list-timers --all

# 수동 실행
sudo systemctl start backup.service

# 로그 확인
journalctl -u backup.service
journalctl -u backup.timer
```

## 15.6 logrotate

### logrotate 설정

```bash
# 기본 설정
cat /etc/logrotate.conf

# 개별 앱 설정 위치
ls /etc/logrotate.d/
```

```bash
# 커스텀 설정 추가
sudo nano /etc/logrotate.d/myapp
```

```
/var/log/myapp/*.log {
    daily               # 매일 로테이션
    rotate 30           # 30개 보관
    compress            # gzip 압축
    delaycompress       # 최신 로그는 압축 안 함
    missingok           # 파일 없어도 오류 없음
    notifempty          # 빈 파일은 로테이션 안 함
    create 644 www-data www-data  # 새 파일 권한
    sharedscripts
    postrotate
        # Nginx에 로그 재오픈 신호
        /usr/sbin/nginx -s reopen 2>/dev/null || true
    endscript
}
```

```bash
# 테스트 실행 (실제 로테이션 안 함)
sudo logrotate -d /etc/logrotate.d/myapp

# 강제 실행
sudo logrotate -f /etc/logrotate.d/myapp

# 전체 실행
sudo logrotate /etc/logrotate.conf
```

## 핵심 요약

```
crontab 빠른 참조:
┌────── 분  (0-59)
│ ┌──── 시  (0-23)
│ │ ┌── 일  (1-31)
│ │ │ ┌ 월  (1-12)
│ │ │ │ ┌ 요일 (0-7)
│ │ │ │ │
0 2 * * * /path/to/script.sh   ← 매일 새벽 2시
*/5 * * * * /monitor.sh        ← 5분마다
0 9 * * 1-5 /weekday.sh        ← 평일 오전 9시
```

- **crontab -e**: 사용자별 cron 편집
- **MAILTO=""**: cron 결과 메일 비활성화
- **스크립트 로그**: cron에서 실행 시 반드시 로그 파일 지정
- **systemd timer**: 로그가 journald에 자동 저장되어 디버깅 쉬움
- **logrotate**: 로그 파일 크기 관리 필수
