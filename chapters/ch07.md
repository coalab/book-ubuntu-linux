# 챕터 7: 네트워크 기초와 설정

> TCP/IP 프로토콜을 이해하고 우분투 네트워크를 설정한다

## 학습 목표
- TCP/IP 프로토콜 스택을 이해한다
- 네트워크 인터페이스를 설정할 수 있다
- 주요 네트워크 진단 도구를 활용할 수 있다
- DNS, 라우팅, 포트를 이해하고 관리할 수 있다

## 7.1 TCP/IP 프로토콜 기초

### OSI 7계층과 TCP/IP

```
OSI 7계층              TCP/IP 4계층
┌─────────────┐       ┌─────────────────┐
│ 7. 응용     │       │                 │
│ 6. 표현     │  →    │ 응용 계층        │ HTTP, FTP, SSH, DNS
│ 5. 세션     │       │                 │
├─────────────┤       ├─────────────────┤
│ 4. 전송     │  →    │ 전송 계층        │ TCP, UDP
├─────────────┤       ├─────────────────┤
│ 3. 네트워크  │  →    │ 인터넷 계층      │ IP, ICMP, ARP
├─────────────┤       ├─────────────────┤
│ 2. 데이터링크│  →    │ 네트워크 접근    │ Ethernet, Wi-Fi
│ 1. 물리     │       │ 계층            │
└─────────────┘       └─────────────────┘
```

### IP 주소 체계

```
IPv4: 32비트, 4옥텟 표기
192.168.1.100 / 24

네트워크 주소: 192.168.1.0
브로드캐스트:  192.168.1.255
사용 가능 범위: 192.168.1.1 ~ 192.168.1.254

서브넷 마스크:
/24 = 255.255.255.0   (256개 호스트, 실사용 254개)
/25 = 255.255.255.128 (128개 호스트, 실사용 126개)
/16 = 255.255.0.0     (65536개 호스트)

사설 IP 대역:
10.0.0.0/8          (대규모 네트워크)
172.16.0.0/12       (중규모)
192.168.0.0/16      (가정/소규모)
```

### TCP vs UDP

| 특성 | TCP | UDP |
|------|-----|-----|
| 연결 방식 | 연결 지향 (3-way handshake) | 비연결 |
| 신뢰성 | 보장 (재전송) | 미보장 |
| 순서 | 보장 | 미보장 |
| 속도 | 상대적으로 느림 | 빠름 |
| 용도 | HTTP, SSH, FTP | DNS, 스트리밍, 게임 |

## 7.2 네트워크 인터페이스 관리

### 인터페이스 확인

```bash
# 네트워크 인터페이스 목록과 IP
ip addr show
ip a               # 축약

# 특정 인터페이스
ip addr show eth0
ip addr show lo

# 인터페이스 통계
ip -s link show eth0

# 구식 명령어 (ifconfig)
sudo apt install net-tools
ifconfig
ifconfig eth0
```

```
ip addr 출력 예:
2: ens33: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP
    link/ether 00:0c:29:12:34:56 brd ff:ff:ff:ff:ff:ff
    inet 192.168.1.100/24 brd 192.168.1.255 scope global ens33
    inet6 fe80::20c:29ff:fe12:3456/64 scope link
```

### 라우팅 테이블

```bash
# 라우팅 테이블 확인
ip route show
ip r              # 축약
route -n          # 구식

# 기본 게이트웨이 확인
ip route | grep default

# 특정 목적지 경로
ip route get 8.8.8.8

# 임시 라우팅 추가
sudo ip route add 10.0.0.0/8 via 192.168.1.1
sudo ip route del 10.0.0.0/8
```

### Netplan으로 네트워크 설정 (Ubuntu 18.04+)

```bash
# 설정 파일 확인
ls /etc/netplan/
cat /etc/netplan/00-installer-config.yaml
```

```yaml
# DHCP 설정
network:
  version: 2
  renderer: networkd
  ethernets:
    ens33:
      dhcp4: true
      dhcp6: false
```

```yaml
# 고정 IP 설정
network:
  version: 2
  renderer: networkd
  ethernets:
    ens33:
      addresses:
        - 192.168.1.100/24
      nameservers:
        search: [local.domain]
        addresses: [8.8.8.8, 8.8.4.4]
      routes:
        - to: default
          via: 192.168.1.1
      dhcp4: false
```

```bash
# 설정 적용
sudo netplan apply

# 검증 (적용 전 테스트)
sudo netplan try
```

## 7.3 DNS 설정

### DNS 동작 원리

```
클라이언트 → /etc/hosts → 로컬 DNS → 상위 DNS 서버
              (우선순위 1)   (캐시)
```

```bash
# /etc/hosts - 로컬 호스트 매핑
cat /etc/hosts
```

```
127.0.0.1   localhost
127.0.1.1   ubuntu-server
192.168.1.10 db.local mydb
192.168.1.20 web.local
```

```bash
# /etc/resolv.conf - DNS 서버 설정 (systemd-resolved가 관리)
cat /etc/resolv.conf

# DNS 서버 설정 확인
resolvectl status

# DNS 조회
nslookup google.com
nslookup google.com 8.8.8.8  # 특정 DNS 서버 사용

dig google.com               # 상세 DNS 조회
dig google.com A             # A 레코드
dig google.com MX            # MX 레코드
dig @8.8.8.8 google.com      # 특정 DNS 사용

# 역방향 조회
dig -x 8.8.8.8
nslookup 8.8.8.8
```

## 7.4 네트워크 진단 도구

### ping - 연결 확인

```bash
# 기본 ping
ping google.com

# 횟수 지정
ping -c 4 google.com

# 인터벌 지정
ping -i 0.5 google.com

# 패킷 크기 지정
ping -s 1000 google.com

# IPv6
ping6 google.com
```

### traceroute - 경로 추적

```bash
# 설치
sudo apt install traceroute

# 경로 추적
traceroute google.com
traceroute -n google.com    # DNS 조회 없이 (빠름)

# tracepath (권한 불필요)
tracepath google.com

# MTR (실시간 traceroute)
sudo apt install mtr
mtr google.com
mtr --report google.com
```

### netstat / ss - 포트와 연결 확인

```bash
# ss (netstat 대체, 더 빠름)
ss -tulnp              # TCP/UDP 리스닝 포트
ss -tnp                # TCP 연결 중인 포트
ss -s                  # 통계 요약

# netstat (구식, net-tools 필요)
netstat -tulnp         # 리스닝 포트
netstat -an            # 모든 연결

# 특정 포트 확인
ss -tlnp | grep :80
ss -tlnp | grep :22
```

```
ss -tulnp 출력:
Netid  State   Recv-Q Send-Q Local Address:Port  Peer Address:Port
tcp    LISTEN  0      128    0.0.0.0:22           0.0.0.0:*     users:(("sshd",pid=1234))
tcp    LISTEN  0      511    0.0.0.0:80           0.0.0.0:*     users:(("nginx",pid=5678))
```

### 포트 스캔과 테스트

```bash
# nmap - 포트 스캔
sudo apt install nmap

# 자신의 서버 스캔
nmap localhost
nmap -p 1-1000 localhost     # 포트 범위
nmap -sV localhost            # 서비스 버전
nmap -A localhost             # 전체 정보

# 특정 포트 연결 테스트
telnet localhost 80
nc -zv localhost 80           # netcat
nc -zv localhost 80-90        # 포트 범위

# curl로 HTTP 테스트
curl -v http://localhost
curl -I https://google.com    # 헤더만
```

## 7.5 네트워크 트래픽 분석

### tcpdump

```bash
# 설치
sudo apt install tcpdump

# 전체 트래픽 캡처
sudo tcpdump -i ens33

# HTTP 트래픽만
sudo tcpdump -i ens33 port 80

# 특정 호스트
sudo tcpdump -i ens33 host 192.168.1.1

# 파일로 저장
sudo tcpdump -i ens33 -w capture.pcap

# 저장된 파일 읽기
tcpdump -r capture.pcap

# 출력 형식 조정
sudo tcpdump -i ens33 -n -v port 443
```

### iftop / nethogs

```bash
# 인터페이스별 대역폭
sudo apt install iftop
sudo iftop -i ens33

# 프로세스별 대역폭
sudo apt install nethogs
sudo nethogs ens33
```

## 7.6 네트워크 본딩과 브리지

### IP 별칭 (가상 인터페이스)

```bash
# 임시로 IP 추가
sudo ip addr add 192.168.1.200/24 dev ens33

# 제거
sudo ip addr del 192.168.1.200/24 dev ens33
```

## 핵심 요약

| 명령어 | 주요 용도 |
|--------|-----------|
| `ip addr` | IP 주소 확인/설정 |
| `ip route` | 라우팅 테이블 |
| `ss -tulnp` | 리스닝 포트 확인 |
| `ping` | 기본 연결 테스트 |
| `traceroute` | 경로 추적 |
| `dig/nslookup` | DNS 조회 |
| `netplan` | Ubuntu 네트워크 설정 |
| `tcpdump` | 패킷 캡처 분석 |

- **Netplan**: Ubuntu 18.04+에서 YAML로 네트워크 설정
- **ss**: netstat의 현대적 대체, 리스닝 포트 확인 필수
- **DNS 우선순위**: /etc/hosts → 로컬 캐시 → DNS 서버
