# 챕터 13: 컨테이너와 Docker

> Docker로 애플리케이션을 컨테이너화하고 배포한다

## 학습 목표
- 컨테이너와 가상화의 차이를 이해한다
- Docker 기본 명령어를 능숙하게 사용할 수 있다
- Dockerfile로 커스텀 이미지를 빌드할 수 있다
- Docker Compose로 멀티 컨테이너 앱을 관리할 수 있다

## 13.1 컨테이너 이해

### 가상화 vs 컨테이너

```
가상화 (VM):                    컨테이너 (Docker):
┌──────────────────────────┐   ┌──────────────────────────┐
│  App A  │  App B  │ App C│   │  App A  │  App B  │ App C│
├─────────┼─────────┼──────┤   ├─────────┼─────────┼──────┤
│  OS A   │  OS B   │ OS C │   │ Libs A  │ Libs B  │LibsC │
├─────────┴─────────┴──────┤   ├──────────────────────────┤
│     하이퍼바이저           │   │      Docker Engine        │
├──────────────────────────┤   ├──────────────────────────┤
│         Host OS           │   │         Host OS           │
└──────────────────────────┘   └──────────────────────────┘
단점: OS 중복, 느린 부팅         장점: 경량, 빠른 시작, 이식성
```

### Docker 핵심 개념

```
이미지 (Image)      → 템플릿 (읽기 전용 레이어)
컨테이너 (Container) → 실행 중인 이미지 인스턴스
레지스트리 (Registry)→ 이미지 저장소 (Docker Hub)
볼륨 (Volume)       → 영구 데이터 저장
네트워크 (Network)  → 컨테이너 간 통신
```

## 13.2 Docker 설치

```bash
# 공식 방법으로 설치
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 또는 수동 설치
sudo apt update
sudo apt install -y ca-certificates curl gnupg

# GPG 키 추가
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 저장소 추가
echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list

sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# sudo 없이 Docker 사용
sudo usermod -aG docker $USER
newgrp docker

# 설치 확인
docker --version
docker run hello-world
```

## 13.3 Docker 기본 명령어

### 이미지 관리

```bash
# 이미지 검색
docker search ubuntu
docker search nginx

# 이미지 다운로드
docker pull ubuntu:22.04
docker pull nginx:latest
docker pull node:20-alpine

# 이미지 목록
docker images
docker image ls

# 이미지 상세 정보
docker inspect ubuntu:22.04

# 이미지 삭제
docker rmi nginx:latest
docker image prune           # 사용하지 않는 이미지 삭제
docker image prune -a        # 모든 미사용 이미지
```

### 컨테이너 실행

```bash
# 기본 실행
docker run ubuntu:22.04 echo "Hello"

# 인터랙티브 (-it), 삭제 (--rm)
docker run -it --rm ubuntu:22.04 /bin/bash

# 백그라운드 실행 (-d)
docker run -d --name mynginx nginx

# 포트 매핑 (-p 호스트포트:컨테이너포트)
docker run -d -p 8080:80 --name webserver nginx

# 환경 변수 (-e)
docker run -d \
  -e MYSQL_ROOT_PASSWORD=secret \
  -e MYSQL_DATABASE=myapp \
  --name mysql-server mysql:8.0

# 볼륨 마운트 (-v)
docker run -d \
  -v /home/student/data:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=secret \
  mysql:8.0

# 리소스 제한
docker run -d \
  --memory="512m" \
  --cpus="1.0" \
  nginx
```

### 컨테이너 관리

```bash
# 실행 중인 컨테이너 목록
docker ps

# 모든 컨테이너 (중지된 것 포함)
docker ps -a

# 컨테이너 시작/중지/재시작
docker start mynginx
docker stop mynginx
docker restart mynginx

# 컨테이너 삭제
docker rm mynginx
docker rm -f mynginx        # 강제 삭제 (실행 중)
docker container prune      # 중지된 컨테이너 모두 삭제

# 실행 중인 컨테이너에 명령 실행
docker exec -it mynginx /bin/bash
docker exec mynginx nginx -t

# 로그 확인
docker logs mynginx
docker logs -f mynginx      # 실시간
docker logs --tail 100 mynginx

# 컨테이너 상태 모니터링
docker stats
docker stats mynginx

# 파일 복사
docker cp file.txt mynginx:/usr/share/nginx/html/
docker cp mynginx:/etc/nginx/nginx.conf ./
```

## 13.4 Dockerfile 작성

### 기본 구문

```dockerfile
# 기본 이미지
FROM ubuntu:22.04

# 메타데이터
LABEL maintainer="student@example.com"
LABEL version="1.0"

# 환경 변수
ENV APP_HOME=/opt/app
ENV NODE_ENV=production

# 작업 디렉토리
WORKDIR $APP_HOME

# 패키지 설치
RUN apt-get update && apt-get install -y \
    curl \
    git \
    && rm -rf /var/lib/apt/lists/*

# 파일 복사
COPY . .
ADD https://example.com/file.tar.gz /tmp/  # URL 또는 tar 자동 압축 해제

# 포트 공개
EXPOSE 3000

# 볼륨
VOLUME ["/data"]

# 실행 사용자
USER appuser

# 기본 명령어
ENTRYPOINT ["node"]           # 변경 불가 명령
CMD ["server.js"]             # 변경 가능 인자
```

### 실전 Dockerfile 예제 - Node.js 앱

```dockerfile
# Node.js 애플리케이션 Dockerfile
FROM node:20-alpine

# 보안: root 대신 node 사용자
WORKDIR /app

# 의존성 파일만 먼저 복사 (캐시 최적화)
COPY package*.json ./
RUN npm ci --only=production && npm cache clean --force

# 소스 복사
COPY --chown=node:node . .

# 사용자 변경
USER node

EXPOSE 3000

HEALTHCHECK --interval=30s --timeout=3s \
  CMD curl -f http://localhost:3000/health || exit 1

CMD ["node", "server.js"]
```

```bash
# 이미지 빌드
docker build -t myapp:1.0 .
docker build -t myapp:latest -f Dockerfile.prod .

# 빌드 캐시 없이
docker build --no-cache -t myapp:1.0 .

# 빌드 ARG 전달
docker build --build-arg NODE_ENV=production -t myapp .

# Docker Hub에 푸시
docker tag myapp:1.0 username/myapp:1.0
docker push username/myapp:1.0
```

## 13.5 Docker Compose

### docker-compose.yml 기본 구조

```yaml
# docker-compose.yml
version: '3.8'

services:
  # 웹 애플리케이션
  web:
    build: .
    image: myapp:latest
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DB_HOST=db
      - DB_NAME=myapp
    env_file:
      - .env
    depends_on:
      db:
        condition: service_healthy
    volumes:
      - ./uploads:/app/uploads
    restart: unless-stopped
    networks:
      - app-network

  # Nginx 리버스 프록시
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
      - ./ssl:/etc/nginx/ssl:ro
    depends_on:
      - web
    restart: unless-stopped
    networks:
      - app-network

  # 데이터베이스
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
      MYSQL_DATABASE: myapp
      MYSQL_USER: appuser
      MYSQL_PASSWORD: ${DB_PASSWORD}
    volumes:
      - mysql-data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped
    networks:
      - app-network

  # Redis 캐시
  redis:
    image: redis:7-alpine
    volumes:
      - redis-data:/data
    restart: unless-stopped
    networks:
      - app-network

volumes:
  mysql-data:
  redis-data:

networks:
  app-network:
    driver: bridge
```

### Docker Compose 명령어

```bash
# 서비스 시작 (백그라운드)
docker compose up -d

# 로그 확인
docker compose logs
docker compose logs -f web

# 서비스 상태
docker compose ps

# 스케일링
docker compose up -d --scale web=3

# 서비스 중지
docker compose stop

# 서비스 중지 + 컨테이너 삭제
docker compose down

# 볼륨까지 삭제
docker compose down -v

# 특정 서비스만 재시작
docker compose restart web

# 컨테이너 내부 명령 실행
docker compose exec web sh
docker compose exec db mysql -u root -p
```

## 13.6 Docker 볼륨과 네트워크

### 볼륨 관리

```bash
# 볼륨 생성
docker volume create mydata

# 볼륨 목록
docker volume ls

# 볼륨 상세 정보
docker volume inspect mydata

# 볼륨 사용
docker run -v mydata:/data nginx

# 사용하지 않는 볼륨 삭제
docker volume prune
```

### 네트워크 관리

```bash
# 네트워크 목록
docker network ls

# 네트워크 생성
docker network create mynetwork
docker network create --driver bridge --subnet 172.18.0.0/16 mynetwork

# 컨테이너를 네트워크에 연결
docker network connect mynetwork mynginx

# 네트워크 내 컨테이너 확인
docker network inspect mynetwork
```

## 핵심 요약

- **이미지**: 불변 템플릿, **컨테이너**: 실행 중인 이미지 인스턴스
- **docker run -d -p 8080:80**: 백그라운드 실행 + 포트 매핑
- **Dockerfile 레이어 캐시**: 자주 변경되는 것은 아래에 배치
- **Docker Compose**: 멀티 컨테이너 앱을 YAML로 선언적 관리
- **볼륨**: 컨테이너 삭제 후에도 데이터 유지
- **HEALTHCHECK**: 컨테이너 정상 여부 자동 확인
