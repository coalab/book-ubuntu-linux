## 학습 목표
- MySQL/MariaDB를 설치하고 보안 설정할 수 있다

## 12.1 MySQL vs MariaDB

| 특성 | MySQL | MariaDB |
|------|-------|---------|
| 개발사 | Oracle | MariaDB Foundation |
| 라이선스 | GPL + 상업용 | GPL |
| 호환성 | - | MySQL 완전 호환 |
| 성능 | 우수 | 일부 쿼리 더 빠름 |
| 추천 환경 | 대기업, Oracle 통합 | 오픈소스 프로젝트 |

## 12.2 MySQL 설치

```bash
# MySQL 설치
sudo apt update
sudo apt install mysql-server

# 서비스 시작 및 활성화
sudo systemctl enable --now mysql

# 버전 확인
mysql --version

# 보안 초기 설정 (필수!)
sudo mysql_secure_installation
```

```
mysql_secure_installation 과정:
1. root 비밀번호 설정
2. 익명 사용자 제거: Y
3. 원격 root 로그인 비활성화: Y
4. test 데이터베이스 제거: Y
5. 권한 테이블 갱신: Y
```

### MySQL 접속

```bash
# root로 접속 (sudo 필요)
sudo mysql

# 또는 비밀번호로 접속
mysql -u root -p

# 원격 서버 접속
mysql -h 192.168.1.100 -u student -p mydb

# SQL 파일 실행
mysql -u root -p mydb < backup.sql

```

### 테이블 관리

```sql
-- 테이블 목록
SHOW TABLES;

-- 테이블 생성
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    is_active TINYINT(1) DEFAULT 1,
    INDEX idx_email (email),
    INDEX idx_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 테이블 구조 확인
DESCRIBE users;
SHOW CREATE TABLE users;

-- 테이블 수정
ALTER TABLE users ADD COLUMN avatar_url VARCHAR(255);
ALTER TABLE users MODIFY COLUMN username VARCHAR(100);
ALTER TABLE users DROP COLUMN avatar_url;

-- 테이블 삭제
DROP TABLE IF EXISTS temp_table;
```

### CRUD 기본 쿼리

```sql
-- 데이터 삽입 (INSERT)
INSERT INTO users (username, email, password_hash)
VALUES ('student', 'student@example.com', '$2y$10$hashedpassword');

INSERT INTO users (username, email, password_hash) VALUES
  ('alice', 'alice@example.com', 'hash1'),
  ('bob', 'bob@example.com', 'hash2');

-- 데이터 조회 (SELECT)
SELECT * FROM users;
SELECT id, username, email FROM users WHERE is_active = 1;
SELECT * FROM users WHERE created_at > '2024-01-01' ORDER BY created_at DESC;
SELECT * FROM users LIMIT 10 OFFSET 20;

-- 데이터 수정 (UPDATE)
UPDATE users SET is_active = 0 WHERE id = 5;
UPDATE users SET email = 'new@example.com' WHERE username = 'student';

-- 데이터 삭제 (DELETE)
DELETE FROM users WHERE id = 10;
DELETE FROM users WHERE is_active = 0 AND created_at < '2023-01-01';

-- JOIN 쿼리
SELECT u.username, p.title, p.created_at
FROM users u
INNER JOIN posts p ON u.id = p.user_id
WHERE u.is_active = 1
ORDER BY p.created_at DESC
LIMIT 10;
```

## 12.4 사용자와 권한 관리

```sql
-- 사용자 목록 확인
SELECT user, host FROM mysql.user;

-- 사용자 생성
CREATE USER 'appuser'@'localhost' IDENTIFIED BY 'secure_password';
CREATE USER 'readonly'@'%' IDENTIFIED BY 'password';         -- 모든 호스트
CREATE USER 'admin'@'192.168.1.%' IDENTIFIED BY 'password';  -- IP 범위

-- 권한 부여
GRANT ALL PRIVILEGES ON myapp.* TO 'appuser'@'localhost';
GRANT SELECT ON myapp.* TO 'readonly'@'%';
GRANT SELECT, INSERT, UPDATE ON myapp.users TO 'appuser'@'localhost';
GRANT REPLICATION SLAVE ON *.* TO 'replicator'@'%';

-- 권한 확인
SHOW GRANTS FOR 'appuser'@'localhost';

-- 권한 적용
FLUSH PRIVILEGES;

-- 권한 제거
REVOKE INSERT, UPDATE ON myapp.* FROM 'readonly'@'%';

-- 사용자 삭제
DROP USER 'olduser'@'localhost';

-- 비밀번호 변경
ALTER USER 'appuser'@'localhost' IDENTIFIED BY 'new_password';
```

## 12.5 백업과 복원

### mysqldump - 논리적 백업

```bash
# 단일 데이터베이스 백업
mysqldump -u root -p myapp > myapp_backup.sql

# 압축 백업
mysqldump -u root -p myapp | gzip > myapp_$(date +%Y%m%d).sql.gz

# 모든 데이터베이스 백업
mysqldump -u root -p --all-databases > all_databases.sql

# 구조만 백업 (데이터 제외)
mysqldump -u root -p --no-data myapp > myapp_schema.sql

# 특정 테이블만 백업
mysqldump -u root -p myapp users posts > partial_backup.sql

# 원격 서버 백업
mysqldump -h 192.168.1.100 -u root -p myapp > remote_backup.sql
```

### 복원

```bash
# 데이터베이스 복원
mysql -u root -p myapp < myapp_backup.sql

# 압축 파일에서 복원
gunzip < myapp_20240115.sql.gz | mysql -u root -p myapp

# 새 데이터베이스로 복원
mysql -u root -p -e "CREATE DATABASE myapp_restore"
mysql -u root -p myapp_restore < myapp_backup.sql
```

### 자동 백업 스크립트

```bash
#!/bin/bash
# /opt/scripts/mysql_backup.sh

BACKUP_DIR="/var/backups/mysql"
DB_USER="root"
DB_PASS="your_password"
RETENTION_DAYS=7
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 모든 데이터베이스 목록 가져와서 백업
for DB in $(mysql -u$DB_USER -p$DB_PASS -e "SHOW DATABASES;" 2>/dev/null | \
           grep -Ev "^(Database|information_schema|performance_schema|mysql|sys)$"); do
    echo "백업 중: $DB"
    mysqldump -u$DB_USER -p$DB_PASS --single-transaction \
              --routines --triggers $DB | \
              gzip > $BACKUP_DIR/${DB}_${DATE}.sql.gz
done

# 오래된 백업 삭제
find $BACKUP_DIR -name "*.sql.gz" -mtime +$RETENTION_DAYS -delete

echo "백업 완료: $DATE"
```

```bash
chmod +x /opt/scripts/mysql_backup.sh

# crontab에 등록 (매일 새벽 2시)
echo "0 2 * * * /opt/scripts/mysql_backup.sh >> /var/log/mysql_backup.log 2>&1" | \
  sudo crontab -
```

## 12.6 MySQL 설정 최적화

```bash
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
```

```ini
[mysqld]
# 기본 설정
user            = mysql
pid-file        = /var/run/mysqld/mysqld.pid
socket          = /var/run/mysqld/mysqld.sock
port            = 3306
datadir         = /var/lib/mysql

# 로그 설정
general_log     = 0                          # 일반 로그 (개발 시만)
slow_query_log  = 1                          # 느린 쿼리 로그
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 1                          # 1초 이상 = 느린 쿼리
log_error       = /var/log/mysql/error.log

# InnoDB 설정
innodb_buffer_pool_size = 1G                 # 전체 메모리의 70-80%
innodb_buffer_pool_instances = 2
innodb_log_file_size = 256M
innodb_flush_log_at_trx_commit = 1          # 안전성 우선
innodb_flush_method = O_DIRECT

# 연결 설정
max_connections = 200
connect_timeout = 10
wait_timeout = 600
interactive_timeout = 600
max_allowed_packet = 64M

# 문자 인코딩
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
```

## 핵심 요약

- **설치 후 필수**: `mysql_secure_installation` 실행
- **사용자 원칙**: root 직접 사용 금지, 앱별 전용 계정 생성
- **백업 전략**: mysqldump + cron + gzip 압축 + 보존 기간 설정
- **innodb_buffer_pool_size**: 전체 RAM의 70-80%로 설정
- **EXPLAIN**: 느린 쿼리 최적화 시 반드시 실행 계획 확인
- **utf8mb4**: 이모지 포함 완전한 유니코드 지원 (utf8 대신 사용)
